> Has anyone out there written/heard of/used an app to stress test a
> server? I need to do some profiling, and I'd rather not have to write
> my own. :) Just something that connects a few clients over and over
> for a while, sends some stanzas, disconnect, do it again, etc.

There is the Tigase Test Suite:
http://www.tigase.org/en/project/testsuite

I use for testing the Tigase server. Here is a test page with
all the automatic tests I run for each server release:
http://www.tigase.org/en/tests

which can give you an idea what it can do.

The suite can be also used to do heavy load tests which I have
run on the Tigase server.
http://www.tigase.org/en/150k-load-tests

And at this address you can find some documentation:
http://www.tigase.org/en/content/tigase-test-suite

This is our internal project and it is not officially supported but
I can offer some help if you want to use it.
Most of the scripts written cover almost whole RFC and a few
XEPs and there are also some scripts which cover Tigase
specific functionality.
