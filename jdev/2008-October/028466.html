Sure, presentations and demos are quite welcome. Our format has been a
bit different for each XMPP Summit. However, the first 4 were quite
small (~15 people). At the XMPP Summit 2 months ago we had 45 people! So
I think we need to plan a bit more for such a large group. For the XMPP
Summit at FOSDEM 2009 I think it would be good to have more hackfests
among smaller groups (e.g., some client developers working together on
Jingle, some other client and library developers working on end-to-end
encryption, some server developers working to test TLS+SASL, etc.). But
we would still have lots of time for presentations and discussion.
