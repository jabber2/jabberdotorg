On Wed, Oct 15, 2008 at 12:59 PM, Tomasz Sterna <tomek at xiaoka.com> wrote:
> I was wondering, whether anyone thought about an easy way of assigning
> an OpenID to a JID.
>
> It would be nice to be able to discover someone's OpenID based on ones
> JID (and get all neat features associated to OpenID from there).
> The preferable way would be without using XMPP queries. OpenID is HTTP
> based, and requiring OpenID supporters to add another protocol support
> could pretty much sink the idea.

I'd like to be able to use my jid everywhere, but the problem with
having a 1-1 association between your JID and OpenID is that it
troubles many people about privacy. If an OpenID points to a web
service describing your identity there are no problems since you can
tune the amount of personal information that is disclosed (e.g. name
and surname and not email for avoiding spam). Instead, if you give
your JID to a 3rd party service or you make it public by publishing it
on a blog you open the path for automatic JID harvesters.

Solutions like this one:
http://openid.xmpp.za.net/ff@jabber.bluendo.com  are nice experiments,
but at large scale they can create big troubles, unless we are so
confident that we can fight spam much (much and even more much) better
than email.
