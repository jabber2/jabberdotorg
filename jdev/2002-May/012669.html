Ah...now I see what you mean...

This "presence list" (I'll be a bit more generic, OK?) could be handled
right now, by using the private server-side storage mechanism (aka
"jabber:iq:private").  There are some possible issues I can see with
doing it this way (such as staying synchronized with the roster).  Oh
yeah, and this assumes you're creating your own client (or customizing
an existing OSS client) (-:

The above would be somewhat specific to your client's implementation. 
If you (and others) believe this is something that should be
standardized, then look into putting together a JEP
(http://www.jabber.org/jeps).   This doesn't mean that clients will
suddenly support this, but it can guide future discussions, and give
client implementors a "frame-of-reference".  Plus, PSA is always looking
for more work to do (j/k)! (-:
