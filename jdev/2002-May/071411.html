Several points:

1. The jabberd 1.4.2 code is not "discarded code from the commercial
server" (I assume you mean Jabber, Inc.'s Jabber Communications Platform).
The codebases are separate and distinct.

2. The jabberd 1.4.2 code is actively maintained. However, almost all
development is now proceeding on the 1.5 code in preparation for a 1.6
release. So the project is alive and well.

3. Development plans for the 1.5 code have been discussed on the jabberd
list (http://jabberd.jabberstudio.org/cgi-bin/mailman/listinfo/jabberd)
and I know that jer and/or temas will be posting a more detailed roadmap
in the next week or two.

Hope that helps,

Peter
