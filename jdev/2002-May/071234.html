Hi,
I'm putting together a limited chat/conferencing app on the J2ME platform.
The J2ME MIDP supports only HTTP and anything besides that (like TCP 5222
connections) is a bonus.

With this, I will put together a small client for the mobile/cellphone, 
and a server.  I won't support the two-way TCP connection, but instead will
use Jabber XML packets over HTTP in an RPC-like way.

Strictly speaking this isn't Jabber, but I'd like to conform to the Jabber
philosophy and best practices.

Any thoughts on this?  Would anyone like to collaborate?
If there is outside interest I'll formalize the project.
