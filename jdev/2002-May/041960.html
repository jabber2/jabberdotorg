Take a look at the "Search/Add User" wizard in RhymBox (
http://www.rhymbox.com ), it uses jabber:x:data also.

As for the XHTML, what would it be used for?
I can see XHTML being used to give some flair to the <description/> fields,
but I don't think Jabber clients should start using XHTML <form/> and
<input/> tags (with all the trouble that comes from it).
Or did you mean clients should use XHTML in their queries?  That could be
useful in certain situations (maybe not for iq:search, but inside an
iq:register for example to give your nickname some markup).

Personally, I think XHTML should not be used inside x:data because the
x:data is all about data gathering, and explicitly NOT about visualising the
questionnaire (JEP-0004, section 2.4).
