On Mon, 19 Jul 2004 15:46:41 -0500, Andrew D. Johnson wrote:

> Is it currently possible to configure jabberd2 so that a message sent to a
> JID that has multiple resources logged in with the same priority will
> arrive at all resources rather than only the most recently logged in
> resource?
> 
> If it's not already possible, Would such a configurable feature be
> considered for jabberd2? Would a patch to jabberd2 that allows for this be
> accepted?

Most Jabber servers in existence try to send to only one resource, and
have various algorithms for determining which resource is the "most
available" (e.g., based on last activity time, etc.). This kind of feature
could be set in server configuration, I suppose, but it might also be
something that each user could configure.

Feel free to send in a patch and the jabberd2 folks might accept it.
