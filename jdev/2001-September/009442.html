On Monday, September 17, 2001, at 07:22 AM, Ben Piercey wrote:

> Anyone know if there are any server limitations that restricts roster 
> sizes? (aside from resource-based
> restrictions like disc space and memory). Things like packet buffer 
> sizes, hard-coded limits, etc...

None that I know of. However, there was some discussion a few months ago 
of the problems that the current server implementation would have in 
scaling up to huge rosters. It sounds like performance would get pretty 
dismal past a few thousand members, due to problems like huge in-memory 
data structures, XML parsing of huge roster lists, etc.

—Jens
