On Mon, 12 Jan 2004 23:29, Jim Parslow wrote:
> Using the jabber:component:accept method
>
> In the jabber.xml I have <archive><service>component</service></archive>
> and it seems to send 2 routed packets through, any idea why?

I was seeing this too, and I rationalised it as one packet being the 
'incoming' packet, and one being the 'outgoing' packet... (with respect to 
the router, a single message sent from one user to another is an incoming 
packet followed by an outgoing packet.)

We had to work around this by comparing each packet to the previous and 
removing duplicates. :-(
