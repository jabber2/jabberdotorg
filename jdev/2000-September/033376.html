Clients only talk Jabber to a Jabber server. The Jabber server can 
act as a mediator between different messaging systems like AIM, 
IRC and even mail systems.

To talk to alternate messaging systems, you must add other 
transport. Thus to be able to recieve message from IMAP or POP, 
someone needs to write a POP or IMAP transport and then you 
must configure it for your mail server and then you can hook your 
Jabber client to the transport. 

You'd need the SMTP transport to send someone a message 
through  email.

BTW I've thought of doing this myself, but just haven't had time. :)
