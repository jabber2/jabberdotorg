On Fri, 22 Sep 2000, David Scott wrote:

> The JabberApplet project, quiet until this time :), is getting close to a
> usable state.  Currently in the process of setting up a SourceForge project.
> The applet has been tested using Microsoft 95, 98, 2000 and NT with Netscape
> and IE browsers (bugs still being worked on).  The applet has also run once
> with:
> - Linux/Netscape (mixed reviews)
> - Mac/IE (works pretty well)
> - Mac/NS (mixed reviews)
> 
> I don't have access to a Mac or Linux machine at this time so if any
> Mac/Linux users would take a quick look at http://java.jabber.org/ and let
> me know if it works for you it would be much appreciated.  If it doesn't
> work please send a quick explanation of the environment/problem and take a
> quick snapshot of the java console sending it to dscott at jabber.com.
> 
> Misc Information:
> a) Roster List Groups - Similar to JabberIM (expandable, corresponding
> status displayed in text)
> b) Roster List names - Friendly names displayed (only one group per JID
> supported)
> c) Subcription Acceptance - When a user subscribes to you <ok> adds user to
> you roster, <cancel> rejects/unsubscribes
> d) Chat Windows - Currently hard coded colors (but that's an accomplishment
> considering AWT doesn't support colored textareas)
> e) Subscription Initiation - not supported yet
> f) Registration - not supported yet
> 
> note: JabberApplet previously called Jabblet on jabber.org projects page.
