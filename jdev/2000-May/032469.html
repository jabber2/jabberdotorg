Hello jdev,

  I´m building a unified messaging system and would like to integrate
  jabber.

  I thought of the follogwing links between the existing System (which
  ist Java/Corba-based) and the Jabber Server:

  Notifications:
  We have a notification Sytem where you can trigger notifications
  (for example if someone sends you a mail) and one could the deliver
  them via Jabber if the user is present in Jabber.

  Offline Messages:
  The System could accept Offline Messages for the user and put them
  into the users "unified" inbox.

  Userbase:
  If a user registers to our Messaging System he/she should
  automatically exist on the Jabber Server. As i don´t like Systems
  with 2 userbases that are autmatically kept in the same state (such
  systems tend to produce incorrect data) id like to use my userbase
  in Jabber. As my system is Corba-Based it woul be no problem to
  access the userbase via Corba.

  As i understand Jabber´s acrhcitecture it should be no Problem to
  integrate these three Interfaces into Jabber but I couldn´t find any
  documentation on how to write server Extensions/Modules. I also
  tried reading the Source but this only helped little.

  We are doing public non-profit research so it would be great if
  someone could help.
