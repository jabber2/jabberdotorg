Along those same lines, it would be cool/useful to have a discovery method 
for existing jabber entities to acquire this list from their server.  It 
would be up to servers on the public network to keep track of eachother by 
remembering who they've done s2s with, and occasionally querying each 
other for these lists, and maintaining an aggregate version at each server 
to be passed along when clients/components asked for it.

Rate-limiting might be useful, as that could easily get out of hand.  Does 
this already exist in a browse/disco capacity?

-Matt
