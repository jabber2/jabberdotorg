I am currently developing my own client right now using a combiantaion
of pure html 4.01 transtional and python. I'm sure many people will
laugh at the soultion I came up with. So, dont flame me. 

I first wrote a daemon using python that is passed the login information
from a web form. The daemon logs in and right now just sets your
pressence to online. Then for each message recived it makes a unique
file from a combo of sender and reciepent names then it then meta-refesh
the user to a cgi-bin perl script that reads the connets of the file and
displays them it meta-refreshs every 3 seconds to keep it updated. 

That is as far as I've gotten so far. I only started 2 days ago.

But here are the plans for sending There will be an html form at the
botom of the conversation in the above perl script that will when filled
in and sent it will call yet a thrid script that will via IPC send the
message to the background script to be sent. Then the logout script will
a shut down the daemon and cleanup the files.

(Yes this is probally a bad way to do it, but I wanted to learn about
jabber and python and this was a gap that I saw needed filled)
