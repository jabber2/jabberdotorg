> The way you should do it is NOT the oob JEP.

Thats not really true, infact the oob JEP specifically states that it can be 
used for this purpose.

> In fact you should rather use the disco feature negotiation
> JEP 0030:
> http://www.jabber.org/jeps/jep-0030.html
>
> and then make a protocol to handshake the h323.

Yes a more generalised VoIP negotiation protocol would be useful, it should 
use JEP-0030 and then JEP-0020 and then JEP-0066 to transfer the VoIP URI.

But there is no reason that JEP-0066 cannot be used as it already specifies 
to communicate an h323, sip or callto URI for initiating a VoIP session.

> The protocol you have created then should become a new JEP.

Not sure a whole new JEP is really needed for this, all it would be doing is 
specifying how to use the existing standard protocols together to intiate a 
VoIP session, no new protocols should be needed since they are already in 
place.
