Hi Heiner!

Heiner Wolf schrieb am 2004-09-27 12:51:14:
> Do you fix it in a way that jabberd14 does not return an error? 
> Or do fix it by disabling inband registration? 

I do not want to remove inband registration (and it is already optional,
as the admin can remove the <register/> element from the configuration
file).

I want to change jabberd14 so that the admin can choice if an error is
returned on the iq get or that the server sends an error as it used to
be. I want it to be the administrators choice and I don't know yet what
the default will be. For a server allowing inband registration, it might
be okay to return an error.

> The question is for LLuna. With LLuna I try to give users a easy non-IM
> style user interface. Part of this is automatic registration. So I need
> inband. If there is no (standardized) inband registration then automatic
> registration at arbitrary servers will be difficult. 

I don't think, that inband registration will go away. We even have
JEP-0077 for this. I guess that it won't be available on commercial
setups of XMPP servers, but I expect inband registration to stay on the
Jabber community servers.
