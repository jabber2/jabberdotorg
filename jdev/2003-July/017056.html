> If you have access to the code for your MUD, you might want to consider
> having your MUD connect directly to a Jabber router, or even better,
> have it implement s2s (dialback) so that it can be on the Jabber network
> on its own.

This is it the question... is it better to connect the mud as client
to a server (but how to map the players to jabber-users?) or to make
the mud a jabber-server on its own?

The other problem is: there are diffent communication types.
on one side the direct "tell" (a msg from user to user) on the other 
"channels" (msg from one user to many others)... a tell is not the problem. 
this is mainly the same as IM. But what about the channels? there have to be
an ability broadcast to many users... especially if there are more then
one server... how to synchronise the servers ?
