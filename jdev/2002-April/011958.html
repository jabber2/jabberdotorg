Jabber Instant Messenger's user registration feature appears to be
broken by the jabber open source server 1.4.2 when the server is
running under an OS other than Linux.  At a minimum, it is broken
under OpenBSD/i386 and Solaris 8/sparc.

The symptom is that the server returns a "406 Not Acceptable" error in
response to the client's IQ GET request.

An example of the broken conversation can be seen here:

	http://blank.org/memory/jim.client.debug.txt

	http://blank.org/memory/jim.server.debug.txt

In all cases, the server is jabberd 1.4.2, compiled with libpth 1.4.0
and the default options taken from the configure script.

Any hints as to the cause of this would be appreciated. :)
