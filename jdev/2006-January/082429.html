On Tue, Jan 24, 2006 at 11:54:43AM +0200, Norman Rasmussen wrote:
> On 1/24/06, Peter Saint-Andre <stpeter at jabber.org> wrote:
> > The long-term answer is to build SPPS into the core server level, but
> > that's not likely to happen very soon.
> 
> As I understand it, if it's not at the core server level, then it's
> not SPPS.  If you're providing pubsub via an external component then
> it's not SPPS.  (as I understand it: the jids needs to match
> identically for it to be SPPS).

That is what I think too. I guess we need some kinde a component
protocol extension which would allow connecting external pubsub service 
as SPPS. Things that need to be solved:

1. redirecting SPPS queries addressed to users to the external
component.
2. providing the external component with roster information.

That doesn't seem much and it would be great if the solution would not
be server implementation specific. This way, fully-featured (not SPPS
only) PubSub implementation could be used with any server. And the
protocol to connect them could be useful for other things too.

Greets,
        Jacek
