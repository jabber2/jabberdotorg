At 16:24 Uhr -0700 14.11.2000, David Scott wrote:
>There's a new JabberApplet client at java.jabber.org.  It works with Windows
>(IE, NS), Linux (NS).  If someone with Mac (IE, NS) could take a look and
>let me know if it works I'd appreciate it.

I tried on my MacOS using NS 4.7.5 and MRJ (apple Java) version 2.2.3 
which is the latest, but still is just JDK 1.1.8

JabberApplet does not run at all, I get an error telling me 
"JabberApplet can't start: class jabber.org.Applet.JabberApplet not 
found". Note: older versions of JabberApplet did load succesfully in 
NS.


Using iCab 2.1 (this is an alternat browser for MacOS, which is 
small, fast, and just insanley great <g>) and the same MRJ, it worked 
better.

The UI is a bit ugly, but I guess that's OK for a development version :)

Menus are not properly working, though. I guess this is due to 
differences in Win/*nix and MacOS in regards of menus: on Mac, you 
normally click on a menu, then *hold* the mouse and do your choice, 
while on other OSes you click once, then the menu opens and you make 
your choice.
Now, the problem is that the Applet only opens menus when I release 
the mouse button. This causes the menu to open, then to immediatly 
close again...

With a bit of tricking I still was able to use the menus (you have to 
be quick in accuratly moving the mouse <g>) and it seems to work 
fine. I have not yet tested sending/rece


When you logoff, on my system the login screen comes back - but with 
username and password still in it! This is bad, at least the password 
should be cleared.

Also, login takes some time (which is ok and normal), but there 
should be a bit more information on the progress (e.g. a progress bar 
or some animation or so)


Another bug: when I resize the "chat" window, it gets moved to location (0,0)
