I would tentatively say that the place is either docs or JAM (jabber as 
middleware).  This may not be "correct" but at least I've been stirring up 
the mud in docs about getting protocols nailed down better.  We've been 
discussing modifications to the jabber protocols in the jam list so that's 
why I mention that list as well.

As far as the traffic, almost everything is in jdev right now.  So my 
suggestion is to use docs for general protocol discussions, lurk on jam, 
and when you really want to get a wide range of developer feedback then 
post to jdev.

I've also been actively pushing to make protocol docs a major focus of the 
new Jabber Foundation so you may want to check that out too (see the 
current thread on the topic in the docs list).
