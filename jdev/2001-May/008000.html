Currently the 'reason' is a text string - it would be really nice if it was
instead an error code for clients and not English-speaking humans to interpret
:-)

Currently stream:error isn't really useful to clients. The majority of the
errors reported to a client are 'Invalid XML', even when it is valid XML but
not a valid Jabber packet. It would be nice to pinpoint down these errors and
give clients something to act on.

-David Waite
