AC> I have found one very unpleasant bug in the ICQ-Transport.
AC> Messages containing russian character "ю" (code 0xFE) are cutted on it.
[skipped]
AC> After short look on sources of ICQ-t, I has found following string there:
AC>  SEPARATOR_CHAR = '\xFE in the file icq-protocol.h.
AC> I guess, this SEPARATOR_CHAR is the reason of strings being cutted.
Yes, it seems You are right - I had this problem about a week ago when I first downloaded Jabber client and tried to work with it. I also analysed ICQ transport code and found this define. The problem is that ICQ protocol is somewhat "buggy" and symbol 0xFE is REALLY used as separator in some types of packets (URL, contacts etc - see ICQv5 protocol for details), though in TEXT packet it isn't a separator. So current code should be modified to use 0xFE as separator only for some types of packets, not for all. I already wrote about this some days ago to Schuyler Heath and to bugs at jabber.org but I don't see any action :-(
