Client Cohesion

The first thing that should be done is build a centralized client
documentation source. Maybe we should expand on the Jabber
Programmer's Guide with sections on implementation details
(psuedocode, tips, etc).

I think that all of us client authors now know what works, and what
doesn't when building a Jabber client. These ideas should be expressed
in some form. Of course, there will be some significant differences of
approach, but showing the options is a good thing.

Ideally, this would all converge into a precise guide on how to build
a Jabber client, with the information translating directly into code.

Going beyond documentation, there is the future possibility of
creating some sort of converged project for a "super client". We all
have our own interpretation of what that is, so maybe it will never
happen. At the very least, it is good to dream and discuss.

Finally, what it all comes down to is getting to know each other
better, as we need to work together to make our Jabber clients the
best instant-messengers available.
