I started watching memory requirements of the 1.4 jabber server.

I started with original dist/final tar distribution,
but then downloaded latest cvs version and got the same results.

My server is RedHat 6.2 (Linux 2.2.14 i686).
I have 128M of memory and 780M of swap.
 
When I login and logout repeatedly as the same or different 
user (it is only roster size dependent) the jabberd process will
grow till it eats all but 3M of the physical memory. It takes
about 5M of swap space and then the grow stops. At that time
jabberd has 23M of memory (16M resident).
 
It shouldn't grow as much, I would say it should return back 
after logout everything it allocated after login, but if the
space remains in jabberd in its memory pool I am OK with it.

Important think is it stops growing, before it brings the 
machine down.

Did anybody else seen similar behavior?
