Hi Silviu,

Having installations run through the IBM Sash Weblication Manager provides
a number of advantages to the user. Dependencies are automatically resolved
and installed; the integrity of the digitally signed installation packaging
is verified; and the user is fully informed about what security
implications installing the weblication will have (for example, whether it
requires access to the file system, sockets, etc.). All of these things are
basically lost with an installation EXE unless we reinvent the wheel every
time.

As for the dependency on Internet Explorer, Netscape 4.x simply does not
provide the extensibility required to make Sash work, and Mozilla/Netscape
6 is not really ready yet.

-Mike
