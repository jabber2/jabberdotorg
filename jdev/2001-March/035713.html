At 7:51 Uhr -0700 03.03.2001, David Waite wrote:
>Generally, DNS names should be used.
>
>Specifically,  Jabberd doesn't use DNS internally for 
>identification, so if the name doesn't match what it is
>expecting things will not work.
>
>So when you do <stream:stream to='208.245.212.100' ...> , it doesn't 
>find the jabber session manager specified
>by
><service id="jabber.org">
>
>Also keep in mind that the 'jabber.org' and '208.245.212.100' cannot 
>be the same thing within JSM - if one
>person connected as test at 208.245.212.100, they will be on a 
>different virtual server than test at jabber.org.
