On Fri, Mar 09, 2001 at 11:39:52AM -0800, Jau-Lung Huang wrote:
> but i think whether i can express what the original native encoding is
> before doing utf-8 encoding process in xml.

Well, sure you can do that. But I dont think it help you much. Eighter your
System IS able to map a Unicode Char to a System char, or it is not. It
won't help the System to know what the original encoding was. In fact it
might be more difficult to have the original encoding than have the unicode
char. For example on my system 99% of all times i will not care if it is
iso8859-1,-15 or -9, cause both of them are idendical. I only have to worry
about some different char.

Well, if you add this info, do it on a vcard/user/clierntinfo level, not on
a message level.
