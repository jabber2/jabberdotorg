It is designed to perform "message filtering" such as automatic
forwarding of messages, automatic replies, etc. All of this is based on
rules that the user can set up. E.g., using Gabber I once set up a rule
that any message sent to stpeter at jabber.com will receive an automated
reply telling the sender that I no longer use that account actively and
please send me a message via stpeter at jabber.org. There are many things
you could do with jabber:iq:filter (deny messages from certain users
etc.). It is supported in the 1.4 server but some clients may not enable
you to set up message filtering rules yet. I know Gabber does and I
assume WinJab does as well.
