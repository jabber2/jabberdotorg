> It would be good if we had standardized methods for embedding 
> presence information in web pages. For example, I'd like to 
> be able to point to the following image file and have it show 
> my presence:
> 
> <img src='http://www.jabber.org/users/stpeter.png'/>
> 
> Granted there are challenges here:
> 
> 1. Needs to be opt-in (no presence leaking)
> 
> 2. User needs a way to choose icon set (if there's a choice)
> 
> 3. Jabber server and web server need to share information
> 
> 4. Would not be dynamic (that would require an xmpp: URI)
> 
> At the jabber.org domain, #3 is a bit of a pain (the web 
> server and jabber server are on different machines), but the 
> challenge seems to be surmountable.
> 
> Other IM systems have this and it's one of those cool 
> features that end users really like. So what's holding us 
> back? What's needed to make this happen?
