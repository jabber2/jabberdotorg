Vinod Panicker wrote:
> RFC 3921 states -
> 
>    Upon receiving initial presence from a client, the user's server MUST
>    do the following if there is not already one or more available
>    resources for the user (if there is already one or more available
>    resources for the user, the server obviously does not need to send
>    the presence probes, since it already possesses the requisite
>    information):
> 
> What if the contact has blocked presence-out for a full jid of the
> user?  Should 3921bis be modified to say that presence probes MUST be
> sent on receiving initial presence from a client?

That seems like an edge case to me, but we'll probably add something
about it to rfc3921bis. I'm not convinced that requiring presence probes
is the answer but we'll discuss it when we work on rfc3921bis.
