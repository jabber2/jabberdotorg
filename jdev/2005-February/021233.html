The sm component has a config option <auto-create/>
that, if turned on, causes the sm to create users when
a session starts.  Here's the comment from sm.xml:

"Enableing this option will make it so that a user
create will be triggered the first time a non-existant
user attempts to start a session. This is useful if
you already have users in an external authentication
database (eg LDAP) and you don't want them to have to
register."
