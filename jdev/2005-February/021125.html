>What is your <html/> bit? Free-form HTML?

Data (meant to be HTML) that will be displayed in an embedded browser. Of course, there are security issues like HTML in email. I use it as a so called "micro homepage" that is part of the user profile sheet. Nice, because they can do more and feel more involved if they have a bit more control over the profile sheet. >It's always a challenge to come up with lists of such things -- people

Yes, I did not do it in my app because it is very much work. But there are communities, which already have these classifications. Sometimes thousands of categories. It is an asset. I was wondering if someone listened and would jump in to take the chance and make Jabber compatible with their classification. So, I hope that someone listens and comes forward. :-)
