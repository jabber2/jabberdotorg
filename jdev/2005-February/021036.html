Hi,

there are implementations with graphics for a whiteboard and HTML for
styled text. What I know (which is very little because there is so much
going on) is that Coccinella whiteboard screenshots
(http://hem.fyristorg.com/matben/) look very impressing and I have
implemented whiteboard functions for LLuna (http://www.lluna.de). There
are clients which support styled text, probably with XHTML, but I don't
know details. 

I guess you will have to browse the JEPs for existing protocols, e.g.
JEP-0071: XHTML-IM for styled text, and the client list for features.
Thanks to Peter's work, we have browsable lists of client and protocols.

Yes, persistent storage is a difficult issue, especially with off the
shelf jabber components. LLuna whiteboard drawings are transient, which
is an euphemism for "I dont care". LLuna relies on the replay of MUC
messages only. The whiteboard protocol is designed to be storable by a
bot, but no bot has been implemented yet.
