hi folks,

I wanted to know the current state of integrating jabber with HTTP
protocol. I think it is very important to support clients behind
firewalls. Anyways, i think the whole problem lies in the fact that HTTP
is a non-persistent protocol. The simple workaround is to use polling at
the client side (i guess jabber.com's webim uses this). However, I worked
on a project in which we developed an application that works by using
blocking HTTP requests at the server side. This technique is generic
enough, it does not rely on HTTP 1.1 persistent connection feature, and
can be applied to Jabber, altho it's a hack ;-).

I was on the WCS mailing list (until it got deprecated;-), and
i think i followed the first release, but since then i have lost track.
There is also something called mod_jabber that i heard about...can anybody
explain how is it different from WCS? Does it provide persistence?
