(regarding the CVS of aim-t from 40 minutes ago)
currently an aim user will see an auto reply for every im sent to a 
jabber user whose status is "away"

these patch files make it so that aim users see the away message only 
once, but it will be sent again if there's a 5 minute lag or more in ims 
sent from the aim user; I think this mimics aim's behaivior pretty 
closely, but whatever.
