> Just wondering how this compares to using ICQ through the AIM Transport?

> the transport and there is no support for P2P messaging.

> In the long term I would have thought there would have thought it'd be

icqv7t uses libicq2000, currently part of the ickle GTK client.  

The latest libicq2000 already does P2P, though icqv7t isn't linked against
that yet.  I think that should be easy?  It is definitely necessary since a
high volume Jabber server passing its messages through AOL's server won't
be appreciated...
