On Sunday December 9 2007, Tomasz Sterna wrote:
> On So, 2007-12-08 at 19:01 +0100, Robin Redeker wrote:
> > User A sends a message to the bare JID of user B which is not subscribed.
> > User B answers to the full JID of user A.
>
> IIRC this full-jid chat binding is discouraged and messages should
> always be exchanged between bare JIDs.
> If you want to hold to a specific conversation thread, use the <thread/>
> item.

I recall that, too, but can't remember where I read it. Is this documented 
anywhere? draft-saintandre-rfc3921bis-04/-05 currently has this to say about 
the topic (section 5.1.1):

"If the message is being sent in reply to a message previously received from 
an address of the form <user at domain/resource> (e.g., within the context of a 
chat session), the value of the 'to' address SHOULD be of the form 
<user at domain/resource> rather than of the form <user at domain> unless the 
sender has knowledge (via presence) that the intended recipient's resource is 
no longer available."
