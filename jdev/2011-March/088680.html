Hello, developers!

My name is Alexander. I am going to make my upcoming proposal about
adding GMail notifications[1] and shared status messages[2] support
into mcabber[3]. Could you evaluate them and say if it will be enough
for a complete GSoC project?

 Usually mail checking process converges to sending request in a cycle
every N minutes and it is often required to get actual information
about mailbox status in time. GMail service allows XMPP client to
receive instant information about changes in mailbox as an XMPP
extension. Though it is not a part of XEP. I am going to add such a
functionality that will allow users to specify FIFO-file to write all
notifications into. In further one can use it for its own needs, e.g.
write a simple script for dzen status bar.

Shared status messages is a way of preserving the equal status/status
message for the all connect clients of a certain account.

[1] http://code.google.com/apis/talk/jep_extensions/gmail.html
[2] http://code.google.com/apis/talk/jep_extensions/shared_status.html
[3] http://mcabber.com/

Thank you!

Regards, Alexander
