> Section 5.1 of RFC 3921 probably answers that question.

Section 5.1 of RFC 3921 talks about presence subscriptions.  I'm
basically asking abt "presence stanzas" - eg, when the user's server
gets presence information of a contact from the contact's server as a
result of a presence probe, when is the user's server expected to
deliver it to the user?  Common sense states that it should deliver it
to the user as soon as it receives the stanza, but what abt presence
states of contacts that are connected to the same server?  Will the
server deliver individual presence stanza's per contact to the user? 
Will the server bunch them together and send them across?  And when
exactly will the server send this?  After initial presence?  Before
initial presence?
