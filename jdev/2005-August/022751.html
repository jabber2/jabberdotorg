Daniel Henninger wrote:
> I don't suppose there is a Uri prefix that indicates a chatroom is there? 
> Or any interest in such a thing?  Basically I think it would be nice if
> there were a link that could trigger the client to pop up an 'enter
> chatroom' interface.  aim has such a thing for it's illustrious public
> chatrooms.  Anyway, Basically I'd love to be able to to something like:
> xmppmuc:pytransports at conference.jabber.vorpalcloud.org
> 
> Any thoughts or does this already exist?

The xmpp: scheme identifies any entity that speaks XMPP, so there is no 
need for a special xmppmuc: scheme (one of the rules of URIs is that 
schemes must not be multiplied beyond necessity).

More here:

http://www.xmpp.org/drafts/draft-saintandre-xmpp-iri-00.html

(version 01 on the way)

And here:

http://www.jabber.org/jeps/jep-0147.html

A smart application that understands the xmpp: scheme would authenticate 
with its configured XMPP server and send a service discovery request to 
the specified JID before launching an interface so that the end user 
knows what kind of entity it is dealing with.
