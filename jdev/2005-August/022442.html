> In this case, yes the flood will be individual presence stanza's that

> information in the RFC of what has to be done about contact presence

> As I was reffering to earlier, what if the contacts are local to the
> server - the server knows that both contact and user are online.

The RFC does not differenciate between local and remote contacts, it says 
you send presence probes to all contacts (no matter where they are), the 
only case where you dont need to send probes is if the user you are logging 
in as already has another resource online, in which case it can use those 
presences.
