> >This got me to thinking: would
> >developers be interested in having the JSF function as a trusted third
> >party for code? If so, it might make sense to develop JOSL 1.1 to cover
> >such code, with the JSF as the licensor (JOSL 1.0 says Jabber Inc. is the
> >licensor but that is a confusing historical artifact). If developers like
> >this idea, I will work to develop a JOSL 1.1 license.
> Is there any reason to have a separate license? Is there any reason NOT 
> to the the GPL for this?

You cannot use GPL sources in non-GPL software. E.g. jabberd14 has been
used in quite some other projects (on only servers) and some of them are
closed-source products (e.g. Tipic's server). This wouldn't have been
possible if jabberd14 would be GPL-only.
