Matthias Wimmer wrote:
|
| You cannot use GPL sources in non-GPL software. E.g. jabberd14 has been
| used in quite some other projects (on only servers) and some of them are
| closed-source products (e.g. Tipic's server). This wouldn't have been
| possible if jabberd14 would be GPL-only.

True ... but that wouldn't apply to a new project, would it? Wouldn't be
in the best interests of the community to have the GPL for all JSF
sponsored projects?

In addition, managing a license with the attendent legal liability may
not be something the JSF wants to be involved in. Should the JSF be
policing how code it owns is used?

Perhaps a choice of JOSL and GPL for projects whose copyrights are
assigned to the JSF would be a better option. The FSF does not accept
(as far as I know) copyrights for code that are not under the GPL.
