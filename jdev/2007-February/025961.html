On Thursday 15 February 2007 15:38, Richard Dobson wrote:
> This sounds just like the spec I drew up quite some time ago back in 2003
> http://www.xmpp.org/extensions/xep-0101.html

Indeed, the two projects are amazingly similar. Moreover, I'm very surprised 
that I never noticed this XEP.

Now, knowing that XEP-101 exists, I don't see the point of developing DIAS 
as a whole new project. Instead, I would like to help developing XEP-101 by 
integrating some ideas that are present in DIAS.

As an example, the IQ request/result dialog and the HTTP dialog, in DIAS, 
are described in different documents, so that other contexts than HTTP can 
be used as well. There are also some details that could make it to XEP-101, 
such as the additional information contained in the ticket result 
(informative text, direct URL), which can be useful, from a security point 
of view.

Richard, if you like the idea of working together on integrating in XEP-101 
some of the concepts that are present in DIAS, feel free to contact me. 
Also, I am relatively new to this, so I don't know if this is generally 
done in private, or publicly via the list.

Best regards.
