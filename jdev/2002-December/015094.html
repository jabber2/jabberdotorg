Exactly but depending on the server implementation (not every server runs
the jabberd) it could be another server, but if it is a jabberd then what
you are saying is correct, although of course the ip address could well have
a dns entry which you try to connect to instead of an IP but you do have to
make sure that when connecting you specify the original domain/server name.

e.g.

C: connect to jabber.org
S: redirects to server2.jabber.org
C: connects to server2.jabber.org, but when connecting uses jabber.org in
the stream, not server2.jabber.org
