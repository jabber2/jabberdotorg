Hello Developers,


Please do _some_ research if the name was used before naming your 
Binaries/Packages.

In Particular, i found a Project named 'pJab' on jabberstudio.org. I
used this as a Name (even same Case-Spelling) for a Perl Jabber Client 
almost 9 months back.

I will not "hunt" this special case further (this will be the only time
i even mention this), since i do not continue developing pJab, since it 
fulfills my needs (as stated at http://pJab.micq.org), but i have to admit 
that i feel minor annoyed.

Such Things will only confuse users.


On another note, i noticed that the Projects i looked at
jabberstudio.org seldom have "Files" listed.

Please be aware that this will block Usage of Jabber, since Users will
assume there is no Release for this Package yet - getting frustrated
after months with no obvious "Release".


Also, it doesnt help anyone if one registers a Project at
jabberstudio.org and then simply ignores it. I kindly ask the
Administrators of jabberstudio.org to check if there is some base that
this Project really will be done before allowing the Project
to be hosted at Jabberstudio.org.
