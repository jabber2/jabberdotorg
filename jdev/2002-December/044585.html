The jabberd project team is pleased to announce the release of jabberd
2.0.0 alpha 3. This is the second in a series alpha releases of the
upcoming jabberd 2 server.

The server is a complete rewrite, with none of the old 1.4 code
included. This makes the transition non-trivial. A future release will
contain documentation and tools for migrating a 1.4 installation to a
2.0 installation.

Not all of the features are in yet, however the system is quite
functional. It should not be used in a production environment yet, as it
hasn't had a lot of testing.

Changes since a2:

    * component presence & service discovery [rob]
    * pipe authenticator [rob]
    * SIGHUP reopens logfiles [rob]
    * Access control for priviliged operations [rob]
    * Support for module multiplexors (eg mod_perl) [rob]
    * Broadcast messages (announce, motd, etc) [rob]
    * Resend messages for SM to admins [rob]
    * IPv6 support [mawis]

Downloads are available here:

  http://www.jabberstudio.org/projects/jabberd2/releases/view.php?id=75

Bug reports and feature requests should be submitted using the tools on
jabberstudio.org. Anything else should be sent to jabberd at jabberstudio.org.

This is also a convenient place to call for contributors. There's still
quite a bit of work to be done, particularly in the area of
documentation. If you'd like to help, take a look at the TODO file, and
drop me a line.

Have fun!

Rob.
