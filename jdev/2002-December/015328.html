To all jabberd developers,
I was trying to run the jabberd-1.4.2 on cygwin, and came across wild 
behavior when I tested the server with over 1000 user, connected online.
I ran it on Win 2000 and win2000 server operating systems with Pentium 4 1.7 
with 256MB.
I was able in the begining to reach this number by changing default winsock 
define of FD_SETSIZE to 2048.
The behavior of the server was as follows:
from about 700 connections, the cpu was rising until, it reached almost 
100%. The memory was ok.
When the server reached 1022 connections, it exited with Segmentation fault.
My questions are:
1. Is there a known usage of the win32 jabberd with many connections?
2. Is it  a known problem - the one described in this mail? or there is a 
known limitation?
3. Is there a patch or solution for this problem? or any ideas how to solve 
it?
4. Is there a native open source port of the jabberd? ( the big problem is 
using the cygwin as a platform for development, which is lacking real 
debugging tools).


Thanks,
Any progress on this issue, will be shared...
