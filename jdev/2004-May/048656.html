According to JEP (http://jabberd.jabberstudio.org/dev/docs/component.shtml)

<quote>
3.5 Sending packets
Packets are sent and received using the <route/> element:
</quote>

but:
<quote>
When a component binds a domain, all the other components currently attached
will receive notice of the fact via a <presence/> packet from the router:

Example 10. Domain advertisement

<presence from='conference.jabber.org'/>
</quote>

<presence/> tag is not within route tags.
So it is not clear, whether all packet shoud be in <route>.
