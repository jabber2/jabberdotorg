Yes, at this point it is recommended that developers compile Iris and their 
programs together as one big heap.  I'll make the library buildable as a DLL 
once I consider the API stable enough.  QCA is a dependency, though you can 
bundle this in statically if you want, like Psi does.

Note: Cutestuff isn't a real dependency.  It's separate in CVS for 
organizational purposes, but a release tarball would contain all the needed 
files together.

Hope that made sense. :)
