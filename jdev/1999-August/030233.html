I'd like to suggest that we kill this thread for the time being.

We agree that the the client and server must both support the
international UTF-8/Unicode and it's compatible with plain ascii, so that
is what will be used by default to and from the server.

In the future we can re-explore supporting other charsets in the server as
needed and as we have installed servers and international clients to test
with.
