> I have a problem with query tag and response tag both being called <info>.
> They should be called diffent things to make the protocol more intuitive.

The queries and responses are processed and routed identically by the
servers, and their overall structure is identical with the exception one
having a <query/> tag and one containing a <response/> tag.  It's that
similarity in structure and treatment by the servers that brought about
using the same tag for both...

One other possibility, which hasn't come up yet, is for an info packet
that is a response to a query, to contain both the response to the sender
and another query for them :)
