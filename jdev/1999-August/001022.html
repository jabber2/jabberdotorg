On 6 Aug 99,, Patrick McCuller sounded off on RE: [JDEV] My evil plans for a clie:

> Because if we're going to haggle the features or non-features or
> configurability or automatic-recompiling of every Jabber client in
> the world, we're all going to get 50000 messages a day and
> subsequently the list will be worthless. 
> 	I strongly suggest that those interested in discussing TCharon's
> or anyone else's Win32 client or the meta-features of meta-clients
> form a list and go there. 

   I think this is an appropriate place to discuss UI -- UI 
suggestions aren't exclusively for the Win32 client, after all, so 
I'd hate to move them to a list for that client. In fact, most UI 
suggestions can be applied to all Jabber clients, whether they be for 
Win32, Java, or what-have-you. Now if someone wanted to start a list 
for Jabber UI discussions, that'd be fine, but in the meantime...

   Also, it's not like the list's gotten high-traffic or something, 
it's only getting like ~30 messages a day. It's not exactly that hard 
to just skip the messages about UI.
