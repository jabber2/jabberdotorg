Now I don't follow the IMPP work (if there has indeed been any yet) but
there is one big issue we must keep an eye on.

If the IMPP standard assumes a system of centralized servers, Jabber will
be more or less incapable of supporting it: which Jabber server will the
other systems send messages to? Of course, we could create a global
gateway server, but that would be very un-Jabber and kind of difficult to
support.

If the issue have already been decided in favour of distributed systems,
please forgive my ignorance, otherwise, we may have a problem.
