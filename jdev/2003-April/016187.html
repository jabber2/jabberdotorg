I noticed some time ago that using a Jabber applet for polling does not
work in any case. I even think it does not work in much more cases then
connecting to port 5222.
The problem is again the sand box of the JVM: Applets can not connect to
the http proxy and try to send the http request (an applet generates)
directly to the http server even if a proxy is configured in the web
browser.

Note: This is only the case of http requests the applet sends. http
requests that are generated by the class loader are using the http proxy
that is configured.
