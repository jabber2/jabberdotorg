You could set up each server to use the alias tag in the c2s section, so
that they all use the one logical domain, but you'd have to use a db
backend like xdb_sql or xdb_ldap so that each person can access their
account from each server. Further, you would not see presence of people
on other servers. Since each server would consider itself to be
jabber.name.com, you could only subscribe to people currently logged
into the same server as you as well. The jabber.org implementation does
not support server farming at this time.
