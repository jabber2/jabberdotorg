The Jabber protocol could probably be used for what you describe, but
it's not going to be very robust compared to a Message Queuing product.
It's also going to be a whole lot of work. 

The jabber protocol doesn't support transactions, doesn't support true
guaranteed delivery, and has no concept of a permanent message store.

You're much better off using an off-the-shelf Message Queuing product.
There are a number on the market that you should be able to use from
both C# and Java (MSMQ springs to mind immediately). If you go with
MSMQ, you could use QueuedComponents to achieve the push functionality
you're looking for. 

I think all the major vendors sell tools such as this (IBM, See Beyond,
Microsoft, etc).
