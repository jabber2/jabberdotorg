On Thu, Apr 10, 2003 at 07:50:04PM +1200, Matt Kearse wrote:
> Hi everyone,
> 
> I am adding Jabber server to server communication support to our own instant messaging
> server and I have a small problem.
> 
> Is there support in the Jabber protocol to run a server on a port other than 5269 for
> server to server communication.
> 
> I am on a LAN which has a single IP address accessible from the internet and would like
> to be able to run more than 1 Jabber server (a test one and a production one).
> 
> I had hoped I would be able to refer to my server using a domain name of the form
> netwin.co.nz:5270 instead of netwin.co.nz, but that doesn't seem to work when talking
> to existing Jabber servers during server dialback.
> 
> Does anyone know if it is possible to do this in some way? Or is there some other
> way around my problem?

You will need third jabber server as router (one of those two can do
this function too). This server would do only s2s - no components, no
clients. AFAIK such router can be linked with servers the same way you
link external components to server.
