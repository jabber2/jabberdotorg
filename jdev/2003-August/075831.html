Yes, this is being reported by a number of people
(http://www.hypothetic.org/docs/msn/phorum/read.php?f=1&i=4434&t=4434).
This could just be some kind of temporary hiccup in authentication
process for MSNP7 and below, or it could be the end of the line for old
protocols.

http://messenger.msn.com/Status.aspx?product=wm reports everything's OK,
which suggests this is forever.  If that's the case, I'm afraid it's
going to be some time before I can get a new MSN transport working - all
my work so far is based on MSNP7, which is significantly different to 8.

If you're interested in seeing an MSN transport in the near future (or
indeed many new transports), the best thing you can do is to start
writing a Jabber transport library in C.
