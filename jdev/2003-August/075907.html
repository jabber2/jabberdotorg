Hi all,

Winfessor has released a beta version of a web service for doing Jabber
communication. It is built on top of our SoapBox Framework, but does not
require .NET or the SoapBox Framework for usage on the client side (just
the server). All you need on the client is a SOAP implemenation that can
handle SOAP Headers. It is meant to serve as a proxy to any Jabber
server. Read more info here:
http://www.winfessor.com/forum/ShowPost.aspx?PostID=124

Documentation does not yet exist so contact me on IM if you'd like more
info on how to use it (since I wrote it) at jconley at jabber.org. Feel
free to send feature requests and any other comments as well. We may
release the source for the web service, but it will still need to run on
top of our proprietary SoapBox Framework, and we haven't worked out
licensing.

JD
