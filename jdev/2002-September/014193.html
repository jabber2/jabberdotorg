Hello all,

  Just a quick notice to anyone using CVS under linux from jabberstudio.org - 
  if you experience cvs hanging during a checkout or update, it could be
  related to your version of cvs. Apparently the latest stable version
  of Mandrake and RedHat both use cvs v1.11.1p1, whereas the latest
  version available is v1.11.2. The problem appears to be a slight
  change to the protocol when using the -z option for compression. The
  command runs correctly, grabbing the requested files, then asks the
  server for final confirmation, which it never gets.

  The solution is to upgrade to v1.11.2, which appears to be in the
  development versions of both distributions. You can also just press
  Control-C if you are sure the syncing has finished, to terminate the
  command. You can also not use the -z option, but not everyone has a
  fast network connection ;)

  Hope this explains things. I've tested on several machines, and the
  upgrade does fix the issue.
