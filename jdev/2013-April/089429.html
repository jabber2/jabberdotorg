Hello,

I am working on an xmpp component (with ejabberd server) that
needs to send subscription requests to gtalk users. In the
presence stanza I include a nickname as per the xep-0172 User
nickname protocol [1]. But the nickname doesn't appear when the
gtalk user receives the invitations (in the gmail web client) and
also when they are accepted. Instead it shows the node part of
the jid.

I understand that this is an extension so google may choose not
to respect the nickname. But I have myself subscribed to a few
bots for which the nickname is displayed correctly.

Does any one know how to tell google to use the nickname instead
of the node in the invitations as well as in roster?

[1]: http://xmpp.org/extensions/xep-0172.html#subscription
