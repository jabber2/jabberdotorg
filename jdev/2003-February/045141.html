Hi everyone,

I would just like to say that the JabberD 2.0 code looks awesome. I have to say that Rob has done an awesome job. 

Anyway, its my perception that a lot of the design is in fact borrowed from Apache in terms of modules and or even how the utilies classes are designed. In many ways the JabberD 2.0 looks like a much leaner version Apache 2.0.

One thing that I think the Apache people have done extremely well esp in the 2.0 release is that they abstracted all (most?) of the platform specific functionality to the APR and APR-util libraries. I've noticed that JabberD has also abstracted some of this non-core functionality out into its util lib. 

However, JabberD's networking code still seems very much a pure Unix / Posix implementation. This makes it difficult to get it to run on Windows w/o the Cygwin layer. Would their be some consideration to migrate some of the JabberD 2.0 code to use APR?
