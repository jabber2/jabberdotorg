Brian,

I am doing something similar in specilized client I am writing.  The way I
am approaching it is to write a server for the organization which handles
all the Jabber traffic as a jJabber client and then a client portion which
handles messages internally based upon certain rules.  To the outside it
looks as if there is one Jabber user - on the inside you can set up whatever
rules and roles are appropriate.
