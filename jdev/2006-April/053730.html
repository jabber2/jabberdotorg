While I'm thinking about it, I noticed that the wildfire people have
added 'Number of Connected User Sessions' and 'Get Presence of Active
Users'.  When you update the JEP, you might want to add some Session
queries in addition the existing User queries.  A short list I can
think of:

4.x. Get Number of Online Sessions
4.x. Get Number of Active Sessions
4.x. Get Number of Idle Sessions
4.x. Get List of Online Sessions
4.x. Get List of Active Sessions
4.x. Get List of Idle Sessions

This seems to slightly overlap with 'Get User Statistics', but I think
the info would be useful anyways.

Re: 'Get User Statistics' surely there should be as many values for
'ipaddresses' as there should be for 'onlineresources'.  In fact if
'status' and 'show' for each resource were included in the results
there SHOULD be an asumption that the ordering of items in each list
matches up.
