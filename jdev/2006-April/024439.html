Question - In many of the examples (eg. Example 74 - Get List of
Active Users, Service Informs Admin of Completion) the field type
attribute has been left out.  Surely the field type should be
specified if it's not 'text-single'?  I assume that it should be
required, so that clients that don't understand FORM_TYPE's will parse
the data correctly.

Observation - In some of the examples (eg. Example 56 - Get Number of
Active Users, Service Informs Admin of Completion) the sessionid is
'get-online-users-num:timestamp' (active vs online), for completeness
it should probably be changed to match the node suffix.
