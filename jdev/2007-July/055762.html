On 7/21/07, Robin Redeker <elmex at x-paste.de> wrote:
> PubSub sounds a bit overkill to me for a component protocol.

Well, I think the idea is that people would like to be able to allow
component a much tighter integration with the user's roster.  PEP is
just one idea that is impossible to add as a component, and adding
into the server core is considered overkill by some.  Being able to
connect a component as a subnamespace on the server's jid would be
very useful.

> I like the approach of Tomasz' protocol very much and will probably
> implement it soon in Net::XMPP2.

I implemented the existing jabberd2 component protocol in xmpp.py, I'd
be happy to add the new features too.

BTW: I take my hat off to Tomasz for getting down and implementing
these enhancements to the component protocol.  XEP-0114 has been
slumbering (well I suppose it's a historical XEP, so it's allowed to),
but there's been no action on the jabberd2 component XEP.  I'm looking
forward to being able to extend 'core server features' by using a
component connection.
