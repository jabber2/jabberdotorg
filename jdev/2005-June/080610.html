I was actually just talking about an idea yesterday that plays off a few
existing ideas.  In fact I'm probably stealing this from someone else.
:)

Someone should build an automated registrar service as a component for
the Public Servers page.

An option would then need to be added to all servers to announce
themselves to this registrar after they start up with presence.  The
registrar would then disco the servers and determine their feature set.
If the server supports a special new feature we create they should be
added to the public server list.  Otherwise they can just be cataloged
for a "total server count" or a server map in Google Maps (as I've seen
a project that is doing this).

We could take this further and create some commands or disco extensions
for the public servers to tell the registrar how many registered and
active users, s2s connections, etc they have.

If the component was out there I'm sure we'd add it to our server as
well...
