Hi all,

  I agree that this shouldn't be included in the MUC spec. My initial
feelings when I saw this is that it would 'clutter' the system. The
little piece of history we return (or not, depending) is to allow a new
participant to be able to catch up quickly with the conversation that
has been taking place, rather than having them keep asking 'whats going
on' 

  Longer term than this example. I would consider the data as an
archive, and whilst it would be nice to have a component that allowed a
user to request data from the archive, there is already a way of doing
this with the browser as well. 

Regards,

  David
