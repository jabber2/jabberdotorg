>All of the jadc2s testing was done before these patches were applied to cvs.

>consumes approx TWICE as much CPU as the jinc CM which were looking to migrate

It should be noted that we are moving to JInc's CM as an interim 
solution until we can either A) fix jadc2s, B) migrate to jabberd 2.0, 
or C) find an alternative open source solution.  While JInc. is being 
gracious enough to grant the JSF a free use license for the jabber.org 
server, we want to make sure that the jabberd server  is improved so 
that it can its role as the reference implementation.
