I'd just like to note that there is a fork of the jabber 1.4.2 code 
that is activily maintained. Lat release was on 2003-04-18 and CVS is 
constantly being updatet. 

http://wpjabber.jabberstudio.org/

It's a fork of the normal code that uses POSIX threading instead of Pth.
 This means you can use dualprocessors and such, and it scales very 
 well. Since scalability is one of the most important goals of the 
 project the maintainer might want to look into any memoryleaks that 
 are found, if they are still in his code as well.
