I proposed earlier having support for a self-backgrounding mode for
jabberd.  This was some healthy debate, but I don't know where the
idea stands with developers.

I would like to update my proposal to "either have a
self-backgrounding mode, or distribute docs or a script example with
the package explaining how to daemonize jabberd".  I think "look at
the RPM file" is a fine interim answer to this FAQ "how do I start
jabberd", but longer term, I feel the jabber.org distribution should
be more cross-platform than to rely on the RPM for a key piece of
administrative knowledge.
