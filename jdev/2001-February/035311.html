>>>>> Todd == Todd Bradley <jdev at jabber.org>

    Todd> I've been wondering if we should split this email list in
    Todd> two lists--one for Jabber developers and one for Jabber
    Todd> administrators.

I'm brand new to the list, but I support this idea.

    Todd> I can't count the number of times someone asks a question on
    Todd> Wednesday, someone else answers it on Thursday, and then by
    Todd> Monday a newcomer asks the same question (and usually gets
    Todd> ignored, unfortunately).

To be fair, from my "just-got-the-thing-running" perspective there are
definite improvements that could be made to the documentation
consistency and organization.  The most popular questions, which are
answered by one of:

  - "That package only works with 1.4."
  - "That package only works with 1.2."
  - "You're confusing the 1.2 and 1.4 codebases."
  - "You need the latest version out of CVS."

are answered pretty much nowhere on the various sites or document.
You _have_ to expect the same questions will be asked over and over
again when this information is not prominently available on
*.jabber.org.  The "technical questions" of the FAQ are not
"technical" in nature.  Having a more aggressively maintained
"technical" faq (answering the four issues above) would cut down on
repeat traffic.  IMHO.
