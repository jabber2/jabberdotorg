> Is there any defined behaviour for non-SASL auth being done twice over
> the same stream?

Some servers are nice and allow you to authenticate again if a temporary
failure of some sort occurred (usually bad username/password). It isn't
really defined. Our implementation allows 3 retries by default in both
SASL and non-SASL.
