>Lots of bugs in PEP server implementations are because the XEP itself is 
>written poorly. It doesn't scale: the idea of keeping resources and 
>features of every user from every server on the planet is completely 
>insane. Don't be surprised if you see memory leaks - they are by design :)

Not insane. Many PEP server implementations are closed systems so the scale
is smaller and even the public servers don't keep resource and features from
every user in every other server. The memory leak in Openfire was just a
bug, not a design, and it has apparently been fixed.

Also, if ejabberd doesn't want to provide proper PEP support then you should
just remove it as a listed feature. A missing feature is better than a
broken one.
