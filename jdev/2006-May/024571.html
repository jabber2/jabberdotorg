> But actually, I think I'd rather have the server push the changes to 
> my phone, instead of my phone having to ask for them.  If I know I 
> want to keep my contacts in sync, it would be better if changes 
> occurred in real-time. ;-)

SyncML allows both entities (server and client) to initiate the 
syncronization. In the typical binding of SyncML for mobile phones (WAP) 
a server starts the syncronization with a WAP push (which is based on 
sending the client a SMS).
With an XMPP binding for SyncML this could be done much more elegant as 
an "XMPP push" is nothing special, just a standard stanza sent by the 
server.
