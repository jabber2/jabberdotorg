On Thu May 25 10:02:01 2006, Ulrich Staudinger wrote:
> Hi there,
> 
> has someone a good solution how to reuse an existing community site 
> with md5'd passwords with a jabber server? Only plain passwords can 
> be accepted by the jabber server, which can be checked against the 
> md5'd passwords from the existing community, but digest login for 
> example, is not possible.
> 
> 
Look at transitioning. I can't recall if that's signalled by XMPP's 
SASL profile or not, but essentially, the sequence is:

a) User attempts to authenticate with DIGEST-MD5.
b) Server says "Can't do that, transition"
c) User reauthenticates using plain.
d) Server updates local database, now allows DIGEST-MD5.
