Hello,
my Jabber client does not support privacy lists but the server does. Now
I keep getting authorisation requests from certain MSN and Jabber users
that are obviously spam. My plan was to block their JIDs entirely so
that they won't bug me ever again. I read the XMPP RFC 3921 [1] section
10.14 (Blocking communication) and set myself a new privacy list that
blocks these JIDs. That works quite fine, but when I log out and back
in, the list isn't active anymore. Even if I set it as the default and
active list in the previous session. IMO the RFC doesn't explicitly say
whether the default list is automatically activated upon every
connection by the server. It looks like it isn't. Should it? Or can I
only reasonably use privacy lists if my client activates them on every
login?

Using customised Psi 0.9.3 and Wildfire 2.6.2 with PyMSN/ICQ transports.

[1] http://www.ietf.org/rfc/rfc3921.txt
