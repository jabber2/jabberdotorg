Yeah, which sounds like a nice idea, but...


> SyncML is an XML based syncronization protocol defined by the Open 
> Mobile Alliance (OMA) and already implemented in most modern mobile 
> phones.

As far as I can tell it requires client state to be held on the 
server. That's a non-starter, IMHO, you wouldn't want to hold state 
for every client of every user on your server, would you? I'm also 
not convinced about how efficient it is, even given that client 
state. If I'm wrong, please let me know (on the list, I'm immune to 
embarrasment about being wrong).
