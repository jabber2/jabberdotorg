Glenn-

External components need to stamp packets with a 'from' address if
something will need to know their address to send messages back to. The
easiest way to do this is to simply stamp all outgoing messages as 'from'
the name of the service, however, this isn't necessarily friendly to
server-to-server traffic.

If responding to a packet received by the component, an alternate solution
would be to simply grab the 'to' address on the packet that came into your
component, and simply assume that it wouldn't have been routed to you if
it was supposed to be.

Finally, you could, also, include the name as some sort of server
configuration option.

FYI- This need exists no matter if your external component is JECL based,
or written in some other language, like Java or Perl.
