Hi!

jabberd14 (up to version 1.4.3) used to generate a <stream:error/>
element if the other side sent a <db:result type='invalid'/> element,
"asking for excuse to use a wrong hostname".

I just thought about this and removed generating a stream error in this
case completely. There is no stream error because it is valid to send a
<db:result type='invalid'/> dialback result. If there was an error it
was either failed dialback or we are really using a wrong hostname - but
in any case there was no error on the stream.

I am now just closing the stream sending </stream:stream>.

Any objections? Anybody thinking jabberd14 should still send an stream
error?
