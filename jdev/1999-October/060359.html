Waleed,
    I agree.  Server should only stamp the GMT time.  The users timezone can be
optional.  When the user has it enabled, it should still be included in the
message rather then the profile for the following reason.  What is the user is
traveling around the world and constantly updating his location?  This could
easily be done with cell phones and GPS in the future.
