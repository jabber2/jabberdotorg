The vcard-temp extension (XEP-0054) supports having URL's for the user avatar 
image in a VCard using the EXTVAL element.
<!-- Photograph property. Value is either a BASE64 encoded
    binary value or a URI to the external content. -->
  <!ELEMENT PHOTO ((TYPE, BINVAL) | EXTVAL)>


I was wondering why doesn't vcard-based avatars extension (XEP-0153) supports 
something similar for dynamic updates i.e. passing url's in place of hash of the 
binary for photo. This provides an easy way for dynamic update of user avatar 
for systems using url to pass across the avatar information. Thoughts?
