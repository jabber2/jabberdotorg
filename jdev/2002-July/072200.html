This is a good idea, although I would change the implemenation to use 
iq:rpc.

The few things that you have pulled out as stats are just... a few 
things.  There are LOTS of things that people will wants stats for, and 
the namespace you are describing is very limiting for future growth.

I'd like to see something that uses iabber:iq:rpc, and a methodology for 
gathering stats.  Like a base function to find out what stats are 
available from a a given thing, and how to get those stats.  List a few 
as an example.  But bear in mind that it needs to scale for the future.
