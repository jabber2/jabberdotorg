Someone running a real AOL client sends a chat message with accented 
characters to someone running a Jabber client. The whole message, not 
just the accented characters in never received at the client.

Running the server in debug shows that the correct packet (with the 
accented characters) is received from the AIM transport. But watching 
the XML debug on the client (Exodus) show the packet is never sent.

Has anyone seen this bug? It can be reproduced using Exodus and a real 
AIM (AOL) client. Entering an accented character in a message on the AIM 
client (using Windows ALT+ or character map) produces no message in the 
client.

Any pointers welcome.
