> Jeremie: i saw on  http://xatp.org/docs/apps.html
> the web servicies: SOAP and WSDL to use with jabber

That is simply a list of things that could possibly run over top of XATP,
it doesn't mean any of those can be used with any other or imply any
relationships between them.

> If im not wrong, with XATP i can use for example the google api with jabber?
> www.google.com/api

Theoretically, you could run both at the same time over an XATP stream,
but XATP is still just an experiment to learn about the characteristics
in that type of framing protocol, not something to be used yet in any
fashion.

If you have any particular questions about XATP, there's a couple lists on
the xatp.org to follow-up to.
