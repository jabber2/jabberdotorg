If this proposal is being seriously considered, how about this: The data
file is formatted as XML/RDF.

Personally, I have doubts that a unified data file for jabber clients
would work. The target environments are quite different, the capabilities
of the clients are quite different, and similar unified file formats have
been miserable failures before (e.g. the windows Registry).
