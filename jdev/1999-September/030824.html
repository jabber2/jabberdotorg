	Jabber's Java projects currently include, to my knowledge, and
being the administrator, I guess I should know, JabberBeans.

	JabberBeans is a client library; it isn't actually a client, just
a tool for building Jabber clients in Java. JabberBeans isn't finished,
true, but it works in interactive mode and has the base protocol
functions: groups, messages, status, etc.

	The JabberBeans API is currently stable. There's nothing to
prevent someone from building an actual client at this point. There is a
serious flaw in JabberBeans right now, but the correction (any day now)
won't change the API.

	Java clients would be particularly welcome!  :)

	Join the java-dev or java-announce mailing lists for more 
information.
