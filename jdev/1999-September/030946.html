Hi,

My take on XML for everything.  From the little I know about XML for
markup, it is easy to see how XML can easily encompass any type of
data packet, with if I remember correctly (I probably don't ;;) )what
was called #PCDATA.  In other words, one could start using any and all
mime types right in your IMs (is that the proper term), since they
could be encapsulated in something like:

<ENCODED type=#PCDATA encoding=rot13>
Guvf vf lbhe frperg zrffntr.
</ENCODED>

This is just rampant speculation, but XML is really perfect for this
application, especially if standards are ever to be set.  Can you see
payphones with jabber capabilities?  I want to get more involved with
the perl side of things, and once that is nicely finished, CGI
integration will be really easy.
