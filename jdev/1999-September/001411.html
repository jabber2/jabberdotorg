okay... what about group non-chat? (a la zephyr or gale for instance)

why not do:
    group/(chat|popup)
    single/(chat|popup)

I would suggets for clients to implement this (at their descretion) as merely
the suggested mode. If the user has no mode set for single then default to
whatever is sent to them as the type. Similar for group, though a client might
wish to allow for group by group settings. (Low traffic groups would go well
as chat-style [little scrolling/loss of messages])

Also, type seems rather a generic, and possibly a bad choice for the attribute
name? Might display or mode be better?
