Info/query allows for remote administration, and one advantage to this is we
can now include basic ACLs for what a user can configure. However, a nice
solution to "how will be do the config.x" file is that we have a
"single-user" mode and config.x is essentially piped into a UNIX pipe
connection the etherx. The connection between etherx and *transport can be a
subset of this single-user mode, allowing for bypassing of the new
configuration information.
