The <priority/> tag for messages is *really* ambiguous :)  In fact, I'm
not even sure what it is really used for, there's no clear purpose for it
but there is a need for a priority system in messages.

The only thing so far that I've intended for priority is that when it is
negative the message is considered bulk and wouldn't be stored offline or
generate errors.

Maybe some predefined ranges for different priorities?
0-1000 is normal/default
1000-2000 is important(sorted to top of list, different color)
2000-3000 is very important(delivered even in away/DND mode, popup in
normal mode)
3000+ is critical messages?

One of the key things that will help for these is when we start making
some progress integrating ICQ and AIM transports, finding out what sort of
priority systems they have and mapping them in... it might be best to just
wait until then to hash this out as it's not a critical peice of Jabber.
