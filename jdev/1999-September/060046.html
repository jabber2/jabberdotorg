while a client cannot be required to comply with anything but the
jabber tansport itself, it would be useful to develop some standards for
voluntary compliance. More specificaly what I'm getting at is for storage of
client information, server name, username, password....

I'd suggest:
	clients store files in a .jabber subdirectory
	of the users home directory

	a file caled either jabber.vars or jabber.rc for storage of simple data
	the file format being variable=value (one per line)

	An environment variable JABBER that is a (semi-/)colon seperated list
	of variable=value 's that overrides file settings
	(and whose use should probably be discouraged)

this would be useful for storage of settings other than server, username, and
password, but to do so we would need a central repository of name,
data-type,and use to prevent steeping on other clients toes...
