  I think the question that comes into my head after reading this thread
is, what protocol?  Wanting to negotiate isn't another 'protocol'.  It's
how can the protocol be used to negotiate and handle a voice/video
stream?

  Doing so using the most generic existing solution means the specific
things, such as the handling of the actual data, be in p2p, client
server, or some mix inbetween, allows ANY data to be run thru it that
has some agreed upon way of doing things.  That's WHY things like mime
types exist.  Thats why your browser can open up netmeeting or anything
else to handle the actual data.
