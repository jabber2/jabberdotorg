On Fri, 16 Sep 2005 17:16, Sander Devrieze wrote:
> I see you use the commands "cat" and "less" in the examples to read
> incoming messages. Isn't the command "tail -f" better to do this?

I would argue that the better solution altogether would be to make it a 
directory with one file for each incoming message.  Creation time on the 
message files is the time the respective message was received.

Then code can register for notification in changes in a directory to monitor 
for new messages.

A really perverse modification would expose all this as a set of Maildir 
folders, for access through Pine. ;-)
