Stephen Pendleton wrote:
> After reading over the presentation, the system looks interesting but what
> are some use cases of the software? Would it be used in the same type of use
> cases that J2EE is?

Yes and no. J-EAI is a platform to centralized and organize the data 
stream between several applications. when data stream are exchanged 
through the Integration platform it is easier to administrate the 
Information system and to see what is flowing through the bus.
It can however be seen as a platform that offer services. you can use 
publish and suscribe to be notified of published data in given channel 
for example. This way, your application can be notified of new invoices, 
so that you can check the stock and order new products for example.

An integration platform is a way to avoid one to one interface between 
applications (this become unbearable when the number of applications 
increases).

I hope this helps,
