PA->> <jabber_config>
PA->> <connection host='jabber.org' port='5222'/>
PA->> <user username='pgmillard' password='xxxx' resource='pgm'>
PA->>     <first>Peter</first>
PA->>     <last>Millard</last>
PA->>     <city>N. Tonawanda</city>
PA->> </user>
PA-> </connection>

I can't see the reason of storing my name or city on the client side.
I usually know this information without the help of the computer :)

Clients only need user's jabber id (which contains hostname, port,
and username) and password. All other data are either client spesific
or can be stored on server. 

If you want to share some platform spesific preferences, using
jabber:iq:info:unix/win/mac/etc namespace is a better solution.

Btw, special clients can use their roster and user information
management systems. And they probably store data on client
side. But since such clients are developed for spesific uses,
we can't demand from them to use a generic configuration file.
