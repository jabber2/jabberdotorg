So, are we in some kind of agreement? A client can optionally read
a config file (.jabberrc or _jabberrc) to get generela jabber config info...
The file is an XML document containg one or more <config> tags. A config looks
like:

<config name=default>
	<host>jabber.org</host>
	<port>8001</port>
	<user>bob</user>
	<nick>bozo</nuck>
	<pass>pass</pass>
</config>

And all tags withing a config are optional.

Also, what about a spec for usernames? Are we at least sure that spaces are
disallowed?
