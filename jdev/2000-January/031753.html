 This is something that confused me a bit too. The relation between
etherxd and jserver are something like between inetd and telnetd. What
actually happens is that etherxd listens for connections on whatever ports
it's told to. When there is a connection on the "jserver" port, then it
sends control to the jserver.so - which is a shared library. 

 Think of it like a faster, neater, more trusting implementation of inetd.
Once etherxd is running, and the etherx.xml mentions jserver, it should be
starting OK.
