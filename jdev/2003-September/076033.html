I'm not sure if this was a general comment, or aimed at me.  In case
it's the latter...

I agree that documentation of code is hugely important for a successful
project, and that making code self-documenting (e.g. with a consistent
style) is part of that.  My complaint is that things like JavaDoc and
Doxygen can never solve the time-consuming, complex problems of
documentation - only offer alternate ways of working that suit some people.

More generally, if the problem is that transport authors face a
thankless painful task, the solution isn't to find another hoop for them
to jump through :)
