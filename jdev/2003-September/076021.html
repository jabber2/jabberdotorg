Without questioning wether Java 1.4.2 is 6-10 times slower than C (wich 
it probably isn't at all ;), you could easily write a transport with 
Java that's way more efficient than the current transports. In other 
words: let's not turn this into a my-programming-language-is-better-
than-yours flamewar. If someone wants to have a go at writing some 
transports in Java cause he thinks it will attrackt more developers, by 
all means, let him try! We'll see wether he's right or not. 

In any case I don't think how expressive a language is, or the opinions 
of other on how fast it is, will detirmine how succesfull such a 
project would be. It's more about the number of people in the community 
with Java skills and the time they're willing to invest into the 
project, and the amount of work already done for them (such as 
networkstacks for the network they're trying to write a transport for).
