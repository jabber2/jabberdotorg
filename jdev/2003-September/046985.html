On Wed, 24 Sep 2003, stephane coste wrote:

> Is there a script/utility to migrate yahoo user files to a new domain name ?

No. The names of the yahoo files are hashed which is a one-way function.
While the transport can look up what file he needs if a request for JID
"user at host" comes in, the reverse is simply impossible.

I submitted a patch for that some months ago that converts the XDB files
on-the-fly to the msn/aim/icq type of XDB by converting the user's file on
the user's log in (this fixes some case sensitivity bugs on the way), but
it has not been integrated in CVS yet (actually, nothing at all has been
done in CVS now for longer time ;-).

Let me know if you want the patch. You can't migrate at once in any case,
you need to wait until the spool files have been converted (= all yahoo-t
users logged in at least once with the patched yahoo-t).
