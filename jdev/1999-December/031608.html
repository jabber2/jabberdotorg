Could you write a quickie description of how to set up the current
transport agent?  Or any transport agent at all, for that matter; the
documentation is a little unclear about how the jabber-server plug-in

[I'd like to write a mail gateway for Jabber, since I'm the type that
 prefers email to instant-messaging.  The idea is that the mail transport
 would bounce all Jabber messages to a mailing address, and mail to (for
 example) jabber+<username>@<jabber-server> would get translated into an
 Jabber IM to the proper person.  Note that only a single mail account
 named 'jabber' is needed to set this up on the server side; the +
 character is a (little-known) comment character within e-mail addresses.
 That said, I'm not finding it easy to understand the current transport
 architecture, and so am somewhat at a loss where to begin coding this
