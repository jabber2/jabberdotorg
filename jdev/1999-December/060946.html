Client Libraries
Win32 Status Report
12/21/1999

Greetings:

I've taken it upon myself to go ahead and let the general JDEV public know
the current status of the client-side libs for Win32. At this point in time,
we have a partially functional ActiveX/COM object which will provide a high
level interface to Jabber functionality. It will provide dual interfaces
(IDispatch and native TypeLib) to the Jabber objects. As such, this will
permit multiple language bindings for the COM objects: VBScript, VBasic, VC,
ASP, Delphi, etc.

Currently we anticipate having a release version of this object set on or
around Jan 1. It should be noted, however, that while this *will* be a
usable version, it's likely that some of the interfaces may change as we
firm up the design. With this in mind, we recommend this library be used for
testing purposes only.

Obviously, this has been a long time in coming, but I believe you'll find
the efforts of Peter Millard (pgmillard) and Keith Minkler (TSBandit) to be
worth the wait. Be sure to tell them "thanks" for all the hard work they've
put into the Win32 clients and this library. Of course, kudos are also in
order for the core developers (esp. Jer) who have put so much work into this
exciting new technology. :)

D.
