j-> Cabbar, the GNOME one is fine. I'm using it no problems, bar it has a
j-> "Resource" field you have to fill in when you log in, which I don't know
j-> what's supposed to be there, but it fails to work if the field is blank.

Resource differentiates the multiple connections of the user. For
example, you can use:
john at jabber.org/kate (cabbar)
john at jabber.org/kateZ (zABBER)
john at jabber.org/php (your jabberified application)
etc..

j-> Cabbar supports "Chat to user" type of stuff. I assume it's just
j-> something you have to add to each client in turn.

Normal messages and chat messages are almost same
at the protocol level. Clients can use message type parameter
to show message in the incoming msgs window or in a special
chat window.
