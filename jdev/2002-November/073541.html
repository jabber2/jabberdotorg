SARADHI wrote:

> How to maintain two instances of jabberd1.4.2 on the same system.
>
> I know we can install the same jabberd server at some other folder in 
> the same system and use another port other than 5222 and give a 
> different domain name when starting the Server.
>
> Is there any way that the same jabber server can be acted as two 
> instances. For example, I need to use a server for my company internal 
> purposes and other for external purposes and no relation between the 
> users of this two different systems.
>
Just use two different configuration files. When you start your jabberd 
instances you can tell the server which configuration to use with the 
"-c <configfile>" flag.
Also check that each instance is only binding to one specific IP address 
and not to all of them. Else the other jabberd is not able to bind to 
his address.
