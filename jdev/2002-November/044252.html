On Fri, Nov 22, 2002 at 10:32:16PM +0000, Matthias Wimmer wrote:
> Hi!
> 
> My suggestion to this problem:
> - Send auto replies only with type='headline'
> - Don't auto reply to messages with type='headline'
> (The alternative would be to add a new attribute or element that marks a 
> message as an auto reply.)

I would strongly advise against using type='headline' for this. Even without
the recent discussion on that type of message, I can say with certainty that
is not meant for this kind of usage.

I can think of two alternatives here:

1) the extra element you suggested, in it's own namespace.
2) making auto replies error conditions.

In the second alternative: not responding to a message of type error seems
natural. A new 'error' code should be defined for this, probably of the
form 2xx, since this error message is in response to a message which
was delivered (and keeping analogy with HTTP status codes).
