  This sounds really like a client functionality issue.  Issuing autoreplies as 
a different message type seems kinda silly.  In general, the clients should 
autosend a reply on a thread that has already had an autoreply.  For clients, 
that dont support the threads, then the thread empty would also be considered 
a 'unique ID'.

  In general, when this occurs, karma will eventually disconnect the user, 
really.  It's an interesting issue, really, becouse it can really bring a 
server to it's knees, 'spec when it happens with announcements on an 
administrative user.
