Hi Rob,

  Sorry for all the chaos this has caused. The original design of
  MU-Conference was that the admin created the rooms using a client like
  Exodus, set them persistent and that the service would then handle all
  the xdb settings for you.

  However it was requested for a script to do this as well, which is
  where the roommaker.pl script comes in. I think that I must be running
  a different version of perl to you, as none of the script errors you 
  reported occured in any of my tests.

  Thanks to a bug report, I have found out why roommaker was making
  rooms that didn't exist - the format of rooms.xml was slightly broken.
  I've just rewritten that piece of code, and incorporated the fixes you
  sent. This code has been commited to CVS, and so should be functional
  (thats what my tests have shown, anyway).

  This should now work for you. Any more problems, just email me and
  i'll get them fixed. 

Regards,

  David
