Hi Casey!

Casey Crabb wrote:

> Auto reply should not be necessary at all in Jabber. Clients should not
> do this. Just set an appropriate away message and people can see why
> you're away, you don't need to send a message saying you're away; this
> only confuses people (and clients).
>
> What are the arguments for having auto reply at all?

Yeah ... in most cases I don't like auto replies either. If I send a 
message to somebody that is not "online" or "chatty" I can see this 
myself, getting back an automated reply is annoying. But:
- there are other cases where auto replies are okay, e.g. the session 
manager that forwarded a message addressed to it to the admin
- there are people that request this feature, therefore there will 
always be Jabber clients that have this feature => we have to let them 
implement this feature in a way that it doesn't harm the whole system.

... BTW ... if we would introduce a new attribute to mark auto replies 
everybody that doesn't want to get them can filter the auto replies. 
(That would be nice too.)
