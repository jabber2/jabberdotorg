Hmm, yes. But personally I am strongly against overloading protocol elements
with other meanings. Using type='headline' for automatic replies is far
from intuitive. Maybe my suggestion of adding a new status code is too,
so the best way to go about this is probably adding a <x/> element
of a specific namespace to hold the information that this message indeed
is an automatic reply, analog to the use of message events.
