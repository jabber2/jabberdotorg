First of all, this is an ALPHA 1 release of jabberd 2.0. There MIGHT be a
beta release by the end of the year. Final release sometime early next
year, if all goes well.

As to modules, the core server will do all the things we expect of a
Jabber server (and more) -- rosters, presence, data storage, etc. However,
current components that are written as internal components (e.g., the
existing gateways, conferencing, etc.) will not run natively with jabberd2
since the internal API is totally different. You will need to run those in
a separate jabberd 1.4.2 process until the components are modified to run
as external components (e.g., David Sutton plans to do this for a later
iteration of mu-conference). But the component developers have time to do
this before jabberd2.0 final is released next year.

Or so it seems to me.
