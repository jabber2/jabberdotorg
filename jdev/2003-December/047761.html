Adrian Blakey schrieb am 2003-12-18 03:52:51:
> checking for res_query in -lresolv... no
> checking for res_query in -lresolv (glibc version)... no
> configure: error: Couldn't find required function res_query
> 
> 
> I guess libresolv.a is not on this machine -- is there a work around?

If you only want to make a local setup with server to server
communications disabled, your workaround would be not to build
"resolver" (Removing this subdirectory from the created Makefile")
and to remove the check for this function in the configure script.

But if you want server to server connections, it might be the easiest
way to install the resolver library. There is no other implementation
for the resolver that does not need this function.
