Despite the slow progress, Gabber 2 has come a long way since its 
initial preview release 6 months ago. It's almost to the point where I 
can honestly recommend it over Gabber 1 (not quite there, but almost). 
Temas and I would appreciate it if Gtk2 and Gnome users could test it 
out and start filling out bug reports, feature requests, etc.

Feel free to grab the latest source code from 
http://gabber.jabberstudio.org/download/

I am currently in the process of beta testing a single package which 
Mac OS X users can install in order to get Gabber 2 and all of its 
dependencies installed... and runnable under X11. If you're interested, 
contact me.

Temas is working on "Project Cranberry", a project to bring Gabber 2 to 
Win32. Don't get too excited... yet. We'll get there. ;)

Be sure to check out our user forums: 
http://gabber.jabberstudio.org/forums/

Gabber 2 requires libsigc++ 1.2, gtkmm 2.0, gconfmm 2.0, and libglademm 
2.0. freshrpms.net has these dependencies for Red Hat/Fedora users.

Gabber 2 *CANNOT* be installed simultaneously with Gabber 1. The second
you type 'make install' you are overwriting your Gabber 1 binary. If
you wish to 'make install' Gabber 2, then be kind to your system and
uninstall Gabber 1 first (otherwise you'll have a lot of useless files
lying around). Should you choose not to 'make install' Gabber 2 (as
would be the wise thing to do, as it is far from a 2.0 still), Gabber 2
will run just fine as ./src/gabber from within the source directory.
Just be sure you 'make install-schemaDATA install-data-local' so that
Gabber's GConf data is properly installed.

Gabber 2 is available in JabberStudio CVS as modules 'jabberoo' and
'gabber'

If you've read this far, I congratulate you.
