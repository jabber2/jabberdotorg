Around two years ago, on 2015-03-29 to be precise, Smack 4.1.0 was
tagged and released. A few days ago I've tagged and released Smack 4.2.0
to Maven Central. Smack is a highly modular and portable open source
XMPP client library written in Java for JVMs (Java SE) and Android.

The 4.2.0 release marks a milestone for Smack, but before I got into the
exciting new features, I'd like to thank the various contributors:

I can not remember when Smack had so many contributors. Thanks everyone
and keep the contributions coming.

The notable changes to Smack 4.2 include support for DNSSEC (thanks to a
previous MiniDNS GSOC project), JIDs as first class citizens by using
JXMPP's JID API, and tons of other improvements, new features and bug
fixes. You can read more in the Smack 4.2 Readme and Upgrade Guide [1]
and the Smack's JIRA release notes [2].

Last but not least, thanks to Bilal Siddiq, Smack now has a logo [3].

Ever wanted to contribute to open source? Are you interested in
XMPP/DNS/DNSSEC? Google gives students the chance to work on open source
projects and get paid for it as part of Google's Summer of Code 2017.
The XSF acts as umbrella organization for projects like Smack and
MiniDNS [4]. Feel free to contact me in the gsoc at muc.xmpp.org if you are
interested in participating or if you want to discuss your own
Smack/MiniDNS related project ideas.

- Florian
