Call for generic pub/sub ideas

The following was posted by jer at jabber.org via the Jabber DevZone web site (http://dev.jabber.org/):



Jabber is being used in ways we've never expected, and continually
being
built into new applications and environments.  Many of these uses
share
common traits with each other and can be accelerated by using a
shared
format for the XML they are exchanging over Jabber.



One of those shared demands is for a generic publish-and-subscribe
mechanism that can work easily with any of the data in the network
of applications and services, not just between IM clients and users.
Jabber already handles this for specific things like user presence,
groupchat, and has some experimental drafts like browsing and the new
conferencing, but there isn't a common generic framework that can be
shared by all these applications.



The list pubsub-jig at jabber.org has been
created for disucssion around the creation of a JIG (and eventually
one or more JEPs).
 At this point we need to have all of the existing methods and any
ideas
or proposals be put forth on the list.  In a week or two there will be
a
meeting to discuss the current state of things, draw up the formal
plans
to have the JIG created, and get some ideas of what direction to head
in.
