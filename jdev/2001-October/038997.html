The hash + sequence values are for zero-k authentication.. Checkout:

http://docs.jabber.org/draft-proto/html/zerok.html

for docs.. Also, if you see <digest/> returned, you should still be able to
authenticate, but 0k auth is a "stronger" mechanism for auth'ing... Winjab
(actually JabberCOM does this) automatically picks the "best" supported auth
type for logging in based on the available types in the iq-result.
