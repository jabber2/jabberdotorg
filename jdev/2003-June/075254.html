A lot of the discussion recently has indicated that the legacy protocols 
are not feature-compatible with jabber.  Perhaps we need to:
a) Clearly define what features each of the existing protocols support. 
 (This might also be useful as a marketting tool)
b) Define which features map directly to Jabber features, and to which 
features they map.
c) Create a JEP for each of the other features that client developers 
can implement for better compatibility (or not bother with if they do 
not intend to support that protocol) and transport developers can use as 
well, so that all ICQ transports have the same behaviour from a client's 
point of view.  These JEPs can then be retired as the legacy IM systems 
die off :)
