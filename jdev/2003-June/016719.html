Perhaps we should adopt the term 'supported client' when talking about a 
client which is recommended by a server administrator for use on that 
server?  It seems to be more descriptive (i.e.  If you have problems 
using this client on my server I will help you, there may also be 
documentation on the server's web site about it, but it is in now way 
endorsed by the JSF, and may not actually be any better than any of the 
others, it's just the one with which I am most familiar)
