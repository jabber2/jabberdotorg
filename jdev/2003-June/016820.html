At the moment clients do the ordering of the roster, for normal contacts 
that's not much of a problem for me but for groups it is.

Psi sorts the groups in an alphabetical way which is far from optimal 
for me, especially with a large roster where some groups fall off the 
list some of the time (even when people that are away are hidden).

IMO it would increase the usability of the roster by quite some extent 
if it was possible to save the order of the groups on the Jabber server. 
Of course the client could save some info on the server that contains 
the order but IMO it would be much nicer if this info could be embedded 
in the groups/roster and if this was standardized in a JEP.

Any reasons why this would be a bad idea? How do you think this should 
be implemented?
