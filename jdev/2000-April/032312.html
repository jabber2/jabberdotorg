I just uploaded the following RPMs to
www.saale-net.de/~weiser/download/jabber:

adns-0.7-1.i386.rpm
adns-0.7-1.src.rpm
jabber-transport-1.0pre2-5.i386.rpm
jabber-transport-1.0pre2-5.src.rpm
libetherx-1.0pre2-1.i386.rpm
libetherx-1.0pre2-1.src.rpm
libjabber-1.0pre2-1.i386.rpm
libjabber-1.0pre2-1.src.rpm
libxode-1.0pre2-1.i386.rpm
libxode-1.0pre2-1.src.rpm
pth-1.3.3-2.i386.rpm
pth-1.3.3-2.src.rpm

They're built for SuSE 6.2 but should compile on any other SuSE oob.
Since I used the RedHat-style of building minimal changes to
installation paths should give perfect RedHat-RPMs.

While building them I noticed that

a.) some packages don't provide a DESTDIR-variable to redirect the
installation into some temporary directory. I therefore simply set
prefix to that dir on 'make install'-time.

b.) the main jabber-transport-package is quite inconsistent in using
the automake-given variables. Where needed for my purposes I replaced
$(prefix)/var by $(localstatedir) and the like. Unfortunately I don't
have the time to finish that.

The jabber-transport-package is incomplete in that it doesn't generate
a new unprivileged user to run the jserver as. So jserver will *run as
root by default*. If anyone is interested, I have a changed script
running locally that uses a jabber user to run the server as.

Also there're no useful package descriptions yet.

Hope I helped a bit. Stay up with the great work!
