On Sat, Dec 16, 2000 at 07:45:17PM +1100, Michael Brown wrote:
> Has this be covered already?  If so, is there a solution?

The Solution is to use eighter a protocol which can be easyly made NAT aware
(by rewriting the content like FTP or IRC-DCC handlers) or to always send
and receive via Servers. If both parties have to connect to a server like
the jabber server, NAT is no issue. Of ourse this is a performance penality,
but it would work. 

We could make this in another connection to a file relay server, we do not
need to handle this inside the messaging protocol. Of course we could also
use traditional protocols like SMTP for that.

What bothers me more is the Server2Server Connections. I managed now to set
one up behind a NAT Gatway, using Port Redirecion, but the Callback stuff
was hard to debug.
