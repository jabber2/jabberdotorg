Hello,

I have switch a while back to using DuckDuckGo as my default search 
engine a while back.

One of the nice feature of the engine is the bang syntax that allows 
searching directly on other site search engine.

You can find details about that bang feature here: 
https://duckduckgo.com/bang

It would be great to have a new syntax to search into the specification 
(RFC and XEP) directly from the search engine.
However, it would require XMPP.org to have that search feature.

For now, it seems the search ULR would be:
http://xmpp.org/search/{{{s}}}

but it seems that search is for now limited to the site (meeting minutes 
and blog post).

I think it would be good to try to expand the reach. I am willing to 
help set that up if needed (If other people like the idea).

Cheers,
