This is handled by the client.

If the client application announces presence (no to address) then the
server will route that to all nodes which are subscribed to it.

However, a client application can send different presence with a
specific to address to different contacts (at their full JID).  In this
case, a server won't route the broadcast presence information there, but
instead only route the presence that was specified for that node.

Any XMPP/Jabber server should support this.  I'm not sure about which
(or if any) clients support it.
