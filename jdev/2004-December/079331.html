Hello
referring to my posts in the members list I want to set up a XMPP 
library working group with the following aims:

1.Create a standard for account usage design (API design, not in 
programming language).

2.Think about ways how the amount of work of an application/client 
programmer can be reduced to the minimum (in example „user's default 
JID“ that will be used by the libraries/new applications 
automatically).

3.Work on a model to integrate XMPP into desktop and OS:
- Find a way to make the „user's default JID“ accessible through this 
new API for existing libraries
- Add standard tools that might come with the libraries to 
setup/register an „user's default JID“ and that matchs smoothly into 
the system control tools of the desktop

For this i setup a wiki at http://relayed.net/wiki/

A model I was already working on is 
http://xcd.jabberstudio.org/xcd-lib.jpg
http://xcd.jabberstudio.org

I just want to make (library) developers work together to make it 
more easy to write simple XMPP applications. Whatever I do not 
automatically expect this to be the must standard.

I only want to discuss this now and therefore I welcome any feedback 
that you might post here:
http://relayed.net/wiki/index.php?Strategy

Please back me with creation of this working group.

Edrin
