Ah, so you mean without the need to run an installation procedure such 
as ./configure --prefix=/path etc?

You can try these to achieve that behaviour with ejabberd:
* http://freshmeat.net/projects/repos/ (only an older version of ejabberd)
* Use the Linux installer and install it locally in a directory (you can 
install it in your home directory for this purpose), then copy this directory 
to the remote server and change the second (and maybe also the last) line in 
the file $YOUR_DIR/bin/ejabberdctl

Do you think one of next things will be useful?:
* a configuration file for ejabberdctl in which you can specify the path and 
other settings,
* a command line option for ejabberdctl to specify the path.
