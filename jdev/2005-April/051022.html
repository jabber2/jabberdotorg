On Tue, Apr 19, 2005 at 07:00:35PM -0400, Zhong Li wrote:
> Does anyone know if allow to use proxy65 transfer voice data, not just
> data file?
> 
> I am trying to think about add voice function on Jabber. The best way
> to do it is transfer voice data by Proxy65. It will be high volume
> consume on it. Does anybody can decide Jabber can support it or not?
> We have to add it on Jabber someway if not this way.

Well, JEP-0065 just helps you establish a bytestream between two
endpoints. What you do with that bytestream is up to you. Right now 
the JEP specifies an optional UDP binding, but proxy65 does not
support that yet, so if you want to use proxy65 then you would need
to send voice data over TCP, which is probably not the best way to
do it.
