> 1. Am I right in assuming that a message can be lost
> somewhere between clients so that the sender thinks
> its delivered but actually isn't?

It can be in practice, but IMO you should assume it
never happens. Losses of messages should not happen,
and if they do, this should be fixed at another level
(not in the protocols themselves).

> 2. Should the protocol specify how to handle a <body>
> in a <message> containing whiteboard data? Should it
> be allowed?

If you don't use bodies, i would personally state that
the sender should not include them, and that it's up
to the client to decide what to do with them if a body
is included.
