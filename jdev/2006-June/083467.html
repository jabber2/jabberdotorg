During the last months I've been hacking jinglepipe, a wrapper around
the libjingle library aimed for clients that don't want to link with
libjingle, but would rather use a pipe to communicate with it.  The
code and the distribution are a bit rough, but I think they are now in
good enough shape for a wider audience.

Find all the details at <URL:http://www.dtek.chalmers.se/~henoch/text/
jinglepipe.html>.  Feedback is very welcome.
