On Thursday 08 June 2006 01:50, Joonas Govenius wrote:
> 2. Should the protocol specify how to handle a <body>
> in a <message> containing whiteboard data? Should it
> be allowed?

Think of the following (*):
1. <message>s can be stored offline (since they can be sent to no resource)
2. <iq>s usually require confirmation/response using the same id

IMO
a) if you want to send static data, you can incapsulate it in a <message>, 
but, AFAIR, not in the <body> itself, but in a separate tag in a separate 
namespace (*)
b) if you want to do dynamic data exchange, you'd better use <iq>s

*) Disclaimer: I haven't read the RFCs lately.
