DJ Adams posted a proposal for a Jabber/SOAP binding on the rpc-jig,
see [1], and the resulting thread.

I have prototype implementations of this for both pocketSOAP [soap
client], and 4s4c [soap server]

One problem you're going to run into, is that the web services engine
in .NET is tied to the .NET HTTP pipeline, you won't be able to
directly call the SOAP engine, instead you'll have to host the HTTP
pipeline, and pump calls in that way.
