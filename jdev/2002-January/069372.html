I'm heavilly using multiple resource connections to jabber server.
So I found some bad behavior in ICQ transport:

It uses each "presence change" from each jabber resource for updating
presence on ICQ.
I think that ICQ-t have to distinguish between resources in relation to
priority.

Here is example of what's bad:

- I log in as first resource HOME - ICQ shows online
- I leave home
- I log in as second resource WORK - ICQ is still online
- At home, my client automaticly set presence to XA because of
inactivity -> ICQ goes N/A
   BUT I'm still ONLINE at WORK!!!!!!
