If you have clusters (or paddocks of servers) within your farm that serve the same users you would ensure that the loss of a single machine doesn't render the users unavailable.

The farm would consist of loosly coupled paddocks, with each paddock serving a set of users. The paddocks would use existing clustering and replication technology to provide fault tolerance for those users, and the farm would be able to serve the whole jabber community.

The other advantage of these changes is that companies can expose a single jabber server to the internet which would allow employees on the road to connect in to the internal jabber system whilst maintaining a single point of exposure for administrators to monitor.

Al.

> > Why not add authentication and message relaying to the S2S protocol.
> >
> > This would give four advantages;
> >
> > 1. Any user could log into any machine and the server would relay the
> > authentication request to the relevant machine to handle authentication.
> >
> > 2. The messages for that user would be relayed to the server they are
> > logged in to and then forwarded on to them.
> >
> > 3. Clusters or farms could be constructed to server a a single jabber
> > community and the load shared between them.
> >
> > 4. This would only involve a change to the S2S protol and servers
> > supporting it (of which there are few), and would leave the C2S protocol
> > unchanged and thus not require any client changes.
