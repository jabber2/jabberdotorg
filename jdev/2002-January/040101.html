On Sat, 2002-01-12 at 05:10, admin at jabber.fsinf.de wrote:
> If you send many messages or long messages in a short time, AOL servers
> complain and you will get that "Talking too fast" error. AIM-t just passes
> this error through, it does not generate it. The reason the error does not
> show up with old AIM-t versions is that they simply ignored/dropped the
> error.

I may not have made it clear in that chat, but the error occured under
very light chat traffic originally. At some point however, (and this was
visible from GnomeICU when using icq-compatibility) the user's presense
switches quickly to "Not Available" with no apparant cause, then
immediately switches back. The user receives presence updates for many
users on his/her contact list. As numbers go, it seems that the number
of presence updates is directly related to the number of "Talking Too
Fast" error messages received immediately afterwards.
