There is one implementation I know of, because I helped write it :-) It
is available commercially from Jabber, Inc., and also up and running at
conference.jabber.com.

The one big difference between the protocol documented at
http://jabber.org/?oid=1630 and this implementation is that the
namespace changed. I did this because 1) "jabber:iq:conference" is
already used by generic conferencing, and made writing clients very
difficult. 2) jcf hasn't yet been approved by the jabber foundation, and
thus shouldn't have a "jabber:" prefix, and 3) because there may be
changes between now and when the spec is finished and approved.

I have a list of TODO items which have been in my head for a while,
including the namespace change. I just put it up @
http://jabber.org/index.php?oid=2390
