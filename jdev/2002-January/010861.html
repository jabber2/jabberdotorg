> > Hey all it's about time for a new development cycle for jabberd. 

My rather vague request: My interest is in jabber as an XML messaging
platform. At work we currently have an infrastructure based around a
network of inn servers which agent-like apps hang off and basically get
fed XML messages by inn and dump output back into the inn network.

I'd like to do something similar based around jabber as the messaging
infrastructure so my request is that I'd like to see jabber compare with
inn for robustness of delivery, message storage, resiliancy, clustering
etc etc. I need to be sure that I can pump an XML message in at one end
and even if parts of the network are unreliable and flakey it will
eventually make it out of the other. At the moment inn/usenet provides
this, but I'm not confident that jabber can. I think theres alot to be
learnt from a system with as much history as inn.
