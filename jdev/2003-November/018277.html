> 1) The MUC protocol should be enhanced to enable media conferencing
> administration, possibly merging text conferencing and media
> conferencing into one piece, allowing a media enabled MUC component.

It would be better to design a separate generic audio conference iniation
and control protocol, and just add an invitation protocol into MUC.

> 2) A stream initiation/feature spec for P2P is needed.

Sure, I havent really read SI properly yet so we will need to investigate if
that is the best solution for setting up a mesh (I think thats the correct
term, i.e. everyone connects to each other) of connections for a p2p
conference or if a more streamlined protocol might be better. It will
probably also need to gather the bandwidth on each node so it can be
calculated how many people you can allow into the conference before it will
either need to change to a server based mode or just not allow any more
people in. We might need either a new p2p streaming protocol spec or we
might need an extra layer on top of SI to make this work, I dunno.
