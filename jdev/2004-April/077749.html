Maybe he thinks about the <flash:stream/> element ...

The flash hack in jabberd14/jadc2s is:
- The client is allowed to immediatelly close the <flash:stream/>
  element (which replaces the
  <stream:stream>...stanzas...</stream:stream> element).
- Zero bytes in the incoming stream on the server are ignored.
- The server terminates every stanza it sents using a zero byte.

And yes ... this is completly against the XMPP spec.
