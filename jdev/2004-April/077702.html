hi, i want to announce the coming release of my XCD, XMPP Client 
Daemon.
What is a XCD? -> XCD is something like a jabber lib with extended 
functionality.
the extended functionality enables it to share existing jabber/xmpp 
sessions between many
programs on the same host. this means one session will be used by not 
only one application
at the same time.
the api/user interface is quite easy and any application should be 
easily developed with it,
i wrote you this letter to get some user/criticial feedback befor i 
release.
please respond to me directly and not to the whole list after having 
looked at
http://skabber.rudbek.com/xcd-hlp/alpha3.zip (win32 binaries)
and for programing informations at
http://skabber.rudbek.com/xcd-hlp/

*please note:
for testing the xcd NetMeeting sample in the .zip you need to
set up a service account in XCD with the resource "XCD/service"


This feedback should be related to the following questions.

1. Do you have special features missing in the XCD that should be 
included?
2. what is your favorite way to load the "default" xmpp sessions to 
the XCD?
   a) is the XMPP Client Daemon Control in windows tray ok for you?
   b) would you like the same application for other platforms or a 
shell application without GUI?
3. are there any special ideas about securing IPC user access at the 
same host? TCP sockets have no auth mechanism.
   (maybe use a small pass(word) for each user and his sessions?)
4. how do you like the project at all?
5. what kind of license do you think of should this project have.

thank you for your time,
Edrin
