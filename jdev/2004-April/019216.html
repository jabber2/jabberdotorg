As I have already mentioned in my most recent email, it might be better to
create a proxy so the client can connect (via the proxy) to almost any
jabber server it wants, all we then need to do is dump the standard
<stream:stream> and develop a simple proxy control protocol to replace it,
to use in directing the proxy into connecting to the appropriate server and
what login credentials to use (the proxy can then handle any complex stuff
like the SASL etc), so using the proxy we can offload onto it the portions
of the protocol that are tuff or impossible to do directly in the flash, but
keeping the rest of the standard protocol.
