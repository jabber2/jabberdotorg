The semantics for this are perhaps not as well-defined as they could be,
but the intent is for this to be used when you initiate a stream to a
host and the host redirects you to another host (i.e., does not continue
the stream negotiation). I hadn't thought of the situation you describe:
redirect during an active session. If we did that, I would think the 
client would need to re-connect (i.e., to the new host).
