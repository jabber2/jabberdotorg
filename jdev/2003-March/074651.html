xdb_file by default caches user data in memory. So until the user's data 
is removed from the cache [s]he will still be able to login. You can 
disable the cache by setting the timeout to 0. Other xdb backends (like 
xdb_sql, probably xdb_ldap) should have a more immediate realisation of 
the change.
