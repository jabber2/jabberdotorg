Hey Folks!

Over the past few months we've had a lot of requests for a ICQ > Jabber user converter. I've been pretty surprised at the number of users that folks have on their ICQ accounts (today someone came into #jabber with over 300 users on their account.. how can those horrible ICQ clients handle that many users??). I'm wondering if anyone would (perhaps a perl wizard??) could take some time to hack something like this together. I believe a lot of people would find this useful. Here's my idea of how it might work:

	- Accept configuration information (ICQ #, Jabber account, location of icqdb, etc.)
	- Connect to Jabber server (user would already have to have registered for the ICQ transport)
	- Do *not* send online presence (don't want people messaging this "stupid" client)
	- Load the icqdb and parse it, prepping for sending in as jabber:iq:roster items
	- Start sending items (is there a rate limit here that would have to be obeyed?)
	- Handle any s10n responses that come back from these new additions
	- Logout

Information about the icq99db format is at:

	http://www.linuxsupportline.com/~pgp/linux/icq/icq99db.html

Anyway.. what do folks think? I know it would be valuable.. does anyone want to tackle it? It would be fantastic to have accessible via a web site... we'd put that on Jabber.org for sure.
