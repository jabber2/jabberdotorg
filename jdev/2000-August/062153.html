> It seems to me that a better approach is something like the .forward file
> under Unix - have a way to set forwarding on your old server for a period
> of time, and then once you've had time to propagate your new address to
> everyone you care to know, remove the old account.

This exists in the 1.1 server (in CVS) it's called mod_filter.  It allows you to setup incoming message filters/rules which allow you do do things like store offline, forward to another jid, auto-reply to the sender, based on rules like resource, from, show, type, body, text, etc...  

NOTE: no client that I know of currently supports mod_filter, but they should, it's a cool feature =}
