James (and all others with Jabba Questions..),

  the win32-jabba module in CVS uses a VERY old version of jabberoo than what is currently available.. If you want to get Jabba to compile "out of the box" you will need to use the "old" version of jabberoo that was used to create Jabba.. It can usually be obtained from my workstation at work (http://jabber.dyndns.org/jabberoo.zip) (unfortunatly, someone has apparently turned off my box at work though, so it won't be available till the next time i go in)

  the old jabberoo doesn't need libsigc++, so that eliminates any problems you are having with that lib.... 

  If you haven't noticed already, the Jabba project hasn't been worked on since early 0.9 server.. I've been more busy with server development, and traveling for jabber.com to continue work on the project (not to mention JabberIM and Winjab using all the wind in my sails) =}

  If anyone out there would like to pick up the project, and you can convert Jabba to use the new jabberoo, then I will be glad to give you CVS access, and hand the baton over..  I'd like to see Jabba live on, but I've run out of the coding time I'd like to have.. =}
