BTW: i thought about a "Cooporate Proxy". Instead of running a server with
the own domain, which will force my users to have more than one jabber id i
would be able to run a cooperate proxy. Everybody is using this proxy as the
jabber server, but the proxy only forwards all messages to the real server.

If the proxy encounters a message to another cooperate user it will deliver
it directly, istead of forwarding it to the real server. This way LAN
Communication keeps local.

Of course there are some issues which needs to be resolved, like avoiding to
open more than one connection from the proxy to the normal server (which
wiull generate a problem for authentication). Also the question is what will
happen if i send a message to another cooperate user who is not online...
delivering it to the upstream server wont keep the message private,
delivering it not to the upstream server mey ause a situation where the
cooperate user is not signing into the proxy and does not get the message...
well.. a lot to think about.

BTW: i think we realy need end-to-end encryption to make such issues a
non-topic, since i donthave tu trust the server anymore.
