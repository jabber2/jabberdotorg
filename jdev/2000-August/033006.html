yes, 
  the thread tag is a client-supported thing.. you will get the same problems if you are talking to a client that isn't echoing the threads.. since aim (as a protocol) doesn't support threaded converstaions (to my knowledge) there is no way to keep track of this... 

  The best way for a client to handle this is to put all messages that come in without a thread tag into the last open converstion window, or into a new window, if none already exist..
