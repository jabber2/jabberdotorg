<br>I read the rfc to mean that there cannot (or should) not be multiple sessions<br>between domains _for the same server_.&nbsp; So one could have a server cluster,<br>with several nodes in the cluster connecting the same 2 domains.  
<br><br>I don't think there's any way for a remote host to reliably determine<br>whether multiple remote connections between the same domains <br>in fact belong to the same logical &quot;server&quot;.&nbsp; Things like nat and big 
<br>multiprocessor machines could break the one server per IP address idea.<br><br>So I read the spec to mean it's up to a developer or administrator to <br>determine what constitutes a distinct server, and that the developer
<br>and administrator should (or must, I forget) allow just one connection per <br>domain pair per server. <br><br>All the same, it's good to not waste resources and minimize the number<br>of connections between one site and another.
