<DIV><SPAN class=299111117-24052001><FONT face=Arial color=#0000ff size=2>May I 
offer a few words of sage advice in this area?&nbsp; If personalbuddy becomes 
very popular and you use the roster/subscription approach, you're going 
to&nbsp;soon start hurting your&nbsp;server (and it darn well better not be MY 
server!).&nbsp; I know because I've done this myself with Jabberbot.&nbsp; 
Jabberbot's roster on Jabber.com is about 10 times the size of the next largest 
user's roster.&nbsp; Whenever Jabberbot tries to log in now, it causes all kinds 
of server mayhem because every presence change causes the server to go read and 
parse rosters of everyone in Jabberbot's roster.&nbsp; This, and the subsequent 
traffic backups, crash the server at Jabber.com (which is JCS, of course, but I 
suspect the open source server would have the same problem).&nbsp; That's why I 
turned Jabberbot off about a month ago.</FONT></SPAN></DIV>
