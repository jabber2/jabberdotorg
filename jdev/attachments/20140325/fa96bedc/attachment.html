Hi Michael,<br>
<br>
When developing a chat client, it is very useful to have your own server to<br>
test against.... If there are any errors in what you send the server will<br>
terminate the connection with little or no indication of what is wrong.<br>
Having your own server allows you to see the server logs and messages and<br>
find out what the issues is.<br>
<br>
We have tried many of the open source XMPP servers and I think the one I<br>
would recommend to develop agains is prosody. (<a href="http://www.prosody.im" target="_blank">www.prosody.im</a>) Very very<br>
easy to set up, they have an active developer and user forum and can be<br>
simply installed on windows, mac or linux.<br>
<br>
For a web based client, I would suggest looking at <a href="http://strophe.im/" target="_blank">http://strophe.im/</a>. I<br>
use the 'C' library rather than the Javascript, however, I am told the<br>
strophe.js is really very good.<br>
<br>
Hope this helps<br>
<br>
Good luck with your thesis<br>
<br>
Mat<br>
