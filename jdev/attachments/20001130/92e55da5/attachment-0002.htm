<P><FONT SIZE=2>I do not remember exactly where (possibly the CVS) but there is sample code </FONT>
<BR><FONT SIZE=2>out there for a JUD, it uses an xml file however, no database backend. It </FONT>
<BR><FONT SIZE=2>also doesn't automatically allow people to search for existing users; users </FONT>
<BR><FONT SIZE=2>have to specifically register with this JUD for their information to show up. </FONT>
<BR><FONT SIZE=2>It works nicely though.</FONT>
</P>

<P><FONT SIZE=2>To make it work with jabber 1.2 do the following</FONT>
</P>

<P><FONT SIZE=2>Go to file main.c</FONT>
<BR><FONT SIZE=2>Approximately line 74 : jud__etherx = ehandler_new(jud.... etc</FONT>
</P>

<P><FONT SIZE=2>Change that line to:</FONT>
</P>

<P><FONT SIZE=2>jud__etherx = ehandler_new_ex(jud_etherx, name, &quot;jabber:component:accept&quot;, </FONT>
<BR><FONT SIZE=2>1234)</FONT>
</P>

<P><FONT SIZE=2>(I don't know how my email mangles that, but it is all on one line, the </FONT>
<BR><FONT SIZE=2>function name changes to ehandler_new_ex, the jabber:server changes to </FONT>
<BR><FONT SIZE=2>jabber:component:accept and an extra parameter is added which is the port </FONT>
<BR><FONT SIZE=2>number you decide to use from now on. In this example I will assume you </FONT>
<BR><FONT SIZE=2>picked 1234.</FONT>
</P>

<P><FONT SIZE=2>compile and prey you are not missing any symbols anywhere :)</FONT>
</P>
<BR>

<P><FONT SIZE=2>In jabber.xml, BEFORE the big &lt;service id=&quot;sessions&quot;&gt; ...</FONT>
</P>

<P><FONT SIZE=2>&lt;service id=&quot;jud.mydomain&quot;&gt;</FONT>
<BR><FONT SIZE=2>&nbsp;&lt;accept&gt;</FONT>
<BR><FONT SIZE=2>&nbsp; &lt;ip&gt;127.0.0.1&lt;/ip&gt;</FONT>
<BR><FONT SIZE=2>&nbsp; &lt;port&gt;1234&lt;/port&gt;</FONT>
<BR><FONT SIZE=2>&nbsp; &lt;secret&gt;anything&lt;/secret&gt;</FONT>
<BR><FONT SIZE=2>&nbsp;&lt;/accept&gt;</FONT>
<BR><FONT SIZE=2>&lt;/service&gt;</FONT>
</P>

<P><FONT SIZE=2>In your /etc/hosts</FONT>
</P>

<P><FONT SIZE=2>127.0.0.1 localhost jud.mydomain</FONT>
</P>

<P><FONT SIZE=2>(ie, &quot;mydomain&quot; has to exist in DNS, but jud.mydomain can be 127.0.0.1)</FONT>
</P>

<P><FONT SIZE=2>Then, in the agents section in jabber.xml</FONT>
</P>

<P><FONT SIZE=2>&lt;agent jid=&quot;jud.mydomain&quot;&gt;</FONT>
<BR><FONT SIZE=2>&nbsp;&lt;name&gt;jud.mydomain&lt;/name&gt;</FONT>
<BR><FONT SIZE=2>&nbsp;&lt;description&gt;My First Userdirectory&lt;/description&gt;</FONT>
<BR><FONT SIZE=2>&nbsp;&lt;service&gt;jud.mydomain&lt;/service&gt;</FONT>
<BR><FONT SIZE=2>&nbsp;&lt;register/&gt;</FONT>
<BR><FONT SIZE=2>&nbsp;&nbsp; &lt;!-- this tag meant that users can register with this agent --&gt;</FONT>
<BR><FONT SIZE=2>&nbsp;&lt;search/&gt;</FONT>
<BR><FONT SIZE=2>&nbsp;&nbsp; &lt;!-- this tag meant this agent is searchable --&gt;</FONT>
<BR><FONT SIZE=2>&lt;/agent&gt;</FONT>
</P>

<P><FONT SIZE=2>You then start jud as follows:</FONT>
</P>

<P><FONT SIZE=2>/pathto/jud -e jud.mydomain -s anything jud.mydomain /path/to/a/database.xml</FONT>
<BR><FONT SIZE=2>(just have an empty file the first time you run it)</FONT>
</P>
<BR>

<P><FONT SIZE=2>In your agents list will now be jud.mydomain. Register with it, then search </FONT>
<BR><FONT SIZE=2>for your own info.</FONT>
</P>

<P><FONT SIZE=2>Worked for me, but ymmv :-))&nbsp; (in jabber 1.2 -&gt; btw, </FONT>
</P>
