<DIV><FONT face=&#23435;&#20307;>For aimtrans, you need to run ./congifure with the switch 
"--with-jabber2".<BR>Then make &amp;&amp; make install.<BR>You now need to 
configure jabber.xml for the AIM transport. Add the<BR>following 
sections.<BR>Add a top level service under the &lt;jabber&gt; 
element:<BR><BR>&lt;service id='aimtrans.my_domain'&gt;<BR>&nbsp;&nbsp;&nbsp; 
&lt;accept&gt;<BR>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 
&lt;ip&gt;127.0.0.1&lt;/ip&gt;<BR>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 
&lt;port&gt;7009&lt;/port&gt;<BR>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 
&lt;secret&gt;test&lt;/secret&gt;<BR>&nbsp;&nbsp;&nbsp; 
&lt;/accept&gt;<BR>&lt;/service&gt;<BR><BR>Under the &lt;agents&gt; element 
uncomment the aim transport and change the jid to<BR>match the service just 
created.<BR><BR>&lt;agent jid="aimtrans.my_domain"&gt;<BR>        &lt;name&gt;AIM 
Transport&lt;/name&gt;<BR>        &nbsp; &lt;description&gt;This is the AIM 
Transport&lt;/description&gt;<BR>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 
&lt;transport&gt;AIM/AOL 
ScreenName&lt;/transport&gt;<BR>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 
&lt;service&gt;aim&lt;/service&gt;<BR>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 
&lt;register/&gt;<BR>&lt;/agent&gt;<BR><BR>You should first test that adns will 
be able to resolve the IP addr of<BR>aimtrans.mydomain. On my system, adns would 
not read from /etc/hosts so I<BR>added an A record to our DNS. In any case, you 
should get the following<BR>output from the adnshost command:<BR><BR>[<A 
href="mailto:foo@bar">foo@bar</A> foo]$ adnshost 
aimtrans.my_domain<BR>aimtrans.my_domain A INET 127.0.0.<BR></DIV></FONT>
