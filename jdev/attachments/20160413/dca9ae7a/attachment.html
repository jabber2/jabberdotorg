<br>
I suggested the following proposal as a feature in Conversations and<br>
Xabber (see <<a href="https://github.com/redsolution/xabber-android/issues/634" rel="noreferrer" target="_blank">https://github.com/redsolution/xabber-android/issues/634</a>>,<br>
<<a href="https://github.com/siacs/Conversations/issues/1802" rel="noreferrer" target="_blank">https://github.com/siacs/Conversations/issues/1802</a>>) - but some people<br>
suggested it would be better to think about this as a part of XMPP<br>
itself and not to add features to the client which are beyond the standard.<br>
<br>
So I do now (with some refinements) ;-).<br>
<br>
I run my own Jabber server (based on ejabberd) and I've been using<br>
Jabber for many years now (with Xabber on my Android devices and Pidgin<br>
and Adium on desktops). Also see <<a href="http://arnowelzel.de/wp/en/about-me" rel="noreferrer" target="_blank">http://arnowelzel.de/wp/en/about-me</a>><br>
if you want to know more about me.<br>
<br>
When I noticed that WhatsApp now also has end-to-end-encryption and even<br>
the EFF says that WhatsApp is "secure" (see<br>
<<a href="https://www.eff.org/de/node/82654" rel="noreferrer" target="_blank">https://www.eff.org/de/node/82654</a>>) I thought that Facebook will<br>
eventually rule the world and either you have use e-mail or WhatsApp<br>
when you want to reach other people... really?<br>
<br>
My dream is to have an alternative to WhatsApp which is *not* depending<br>
on a single provider (like WhatsApp, Threema or Signal) but which is<br>
also as easy to install and use as WhatsApp, Threema etc..<br>
<br>
What makes WhatsApp so popular?<br>
<br>
Well - you don't need to create any account using a username/password<br>
and you don't need to know the account names of your friends - their<br>
phone numbers are enough to check, if they use WhatsApp as well and to<br>
reach them. And as you already have the numbers in your phonebook you<br>
literally can start to chat with your friends instantly after installing<br>
WhatsApp.<br>
<br>
How to achieve the same goal with XMPP/Jabber?<br>
<br>
My idea:<br>
<br>
1) Offer to enter an existing account OR to create an account using one<br>
of several well known Jabber providers (maybe the ones listed on<br>
<<a href="https://xmpp.net/directory.php" rel="noreferrer" target="_blank">https://xmpp.net/directory.php</a>>) with a password which is automatically<br>
created locally and which can be displayed or saved for later reference<br>
OR can manually be set.<br>
<br>
2) Detect the local phone number AND/OR allow the user a way to enter<br>
and confirm it manually.<br>
<br>
3) Provide a way to send the hash of the telephone number together with<br>
the Jabber-ID to a central registry OR the selected server which takes<br>
care to handle this data to be synchronized with some kind of central<br>
registry.<br>
<br>
4) Offer a function which allows to find other users by the hashes for<br>
the existing phone numbers in the addressbook at the central registry OR<br>
by asking the selected server which will then return the Jabber-IDs for<br>
the given hashes if there are any known.<br>
<br>
<br>
I'm aware, that the central registry may be seen as "evil" - but this<br>
should be implemented as open source and should be run by a non profit<br>
organisation. But technically it is not possible to know if and where<br>
other people have an account with some kind of registry.<br>
<br>
To make this clear: the communication itself is still possible without<br>
any central registry. The registry is only needed to find others more<br>
easily, not for the communication itself. Of course you still can<br>
contact Jabber-IDs directly without using a registry server. But in fact<br>
people expect that they can use a messenger without first asking all<br>
their friends for some kind of ID - they already have an ID: the phone<br>
number. So it's just a logic decision to use that for messaging as well.<br>
<br>
It's only a very rough proposal and there may still be things I forgot.<br>
<br>
Did anyone try this yet?<br>
