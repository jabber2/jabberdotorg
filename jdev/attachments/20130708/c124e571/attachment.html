>>> *What is it?*<br>
>>><br>
>>> XMPP-FTW exposes XMPP over a websocket* and allows users to interact<br>
>>> using named 'events' and JSON. The server itself sits between the<br>
>>> browser and an XMPP server and translates messages on-the-fly. Events<br>
>>> allow the server to fill in gaps which allows the JSON messages to<br>
>>> only include the parts they need.<br>
>>><br>
>>> Theoretically it is also able to run in a browser and talk to an XMPP<br>
>>> server over BOSH/websocket but I haven't tried to do this yet.<br>
>>><br>
>>> *Who is it for?*<br>
>>><br>
>>> Web-based developers who would like to create XMPP-powered<br>
>>> applications for the browser but are put off by things like XML, BOSH,<br>
>>> and the complexity of building stanzas.<br>
>>><br>
>>> *Website / Source code*<br>
>>><br>

>>> source code is all Apache 2.0 licenced and hosted on github (and<br>
>>> available though node.js's NPM).<br>
