<pre dir="auto" class="k9mail">After years of development I am stoked to announce the release of<br>python-omemo - An open Python implementation of the OMEMO Multi-End<br>Message and Object Encryption protocol!<br><br><br>python-omemo [1] is an OMEMO implementation written from scratch with<br>support for both the currently widespread eu.siacs.conversations.axolotl<br>version of OMEMO [2] and the newest, SCE-based version under the<br>urn:xmpp:omemo:2 namespace [3].<br><br><br>Notably, python-omemo is an implementation of the OMEMO _protocol_ and<br>not just the cryptographic primitives, which means that many of the<br>small protocol flow details of the OMEMO specification are handled by<br>python-omemo as well.<br><br><br>Key properties of the library:<br><br>- Seamless support for both versions of the OMEMO specification at the<br>same time. The library transparently handles compatibility between the<br>two versions, i.e. identity keys, device ids and trust are shared<br>appropriately. The protocol version to encrypt a message with can either<br>be selected explicitly, or given as a priority, to e.g. use the new<br>OMEMO if available, and fall back to the old OMEMO if not.<br><br>- Handles bundle and device list, making sure the published data is<br>always in sync with the local data.<br><br>- Handles small protocol flow details, like sending automated empty<br>messages in response to a key exchange or to prevent staleness.<br><br>- Provides an easy way to integrate any sort of trust system - be it<br>simple manual trust, BTBV, TOFU or something more complex.<br><br>- Thin and straight-forward public API - OMEMO doesn't have to be<br>complicated :)<br><br>- Optionally reads/verifies and builds ElementTree XML structures for<br>you, so that you can keep your codebase even thinner.<br><br>- Clean, modern, fully-typed Python 3 with thorough API documentation.<br><br>- Very easy to get started with - provide a simple key-value storage,<br>implement some methods to upl
