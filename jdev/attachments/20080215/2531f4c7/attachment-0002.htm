As I understand it from one of the Android team members, they have been<br>
using XMPP communications (sent only via the Google Talk service) as a<br>
way for developers to debug their code by sending handset-to-handset<br>
messages. This usage was quite limited and the server was hardcoded to<br>
<a href="http://talk.google.com" target="_blank">talk.google.com</a> for that reason. This package was never intended for<br>
broader use, e.g. as a general messaging service that could be used for<br>
all the things we&#39;re familiar with from the XMPP universe (presence, IM,<br>
RPC, Jingle, etc.). Naturally if developers want a more robust messaging<br>
service that they could hook up to their own XMPP servers, they could<br>
simply use a full XMPP library (a likely candidate is Smack, since<br>
Android is all Java all the time).<br>
<br>
Since Android was never doing anything very extensive with XMPP, it&#39;s<br>
not exactly the end of the world that they are more clearly labelling<br>
this package as a GTalk-only debug channel.<br>
<br>
Now, whether it would be better for Android in the long run if they were<br>
doing something more extensive with XMPP is another question... :)<br>
