class=234414721-16032000>&nbsp;&nbsp;&nbsp; I have made several attempts to port 
the server to win32.&nbsp; currently, there are problems with pth (the threading 
library) porting to win32 properly.&nbsp; I have been working with The lead 
developer of pth to correct this problem, but it doesn't seem that it will be 
solved any time soon.&nbsp; The other option we've looked at was writing a 
native win32 server, but that will definitly not happen until at least 1.0, if 
not later... the reason for this is simple.. rewriting a native win32 server 
creates two build paths.. one for *nix, and one for win32, since features are 
being developed every day for *nix, those features would need to be coded into 
win32 as well, it is much more feasable to wait until the build process has 
slowed down (1.0) to start on a native win32 server, and even then, a direct 
port would be *MUCH* better in the long run.</SPAN></FONT></DIV>
