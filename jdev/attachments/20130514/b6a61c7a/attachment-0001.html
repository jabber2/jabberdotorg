<p dir="ltr">There's two cases I can think of.</p>
<p dir="ltr">1) User has message delivered (ie, the message is not lost). When user focuses the chat window, or otherwise takes action such that the message has clearly been presented to the user, send a receipt. This indicates to the sender that the message has reached the user.</p>

<p dir="ltr">This is useful in the sense that it aids conversation flow - like those messages you send to someone immediately after "Right, back later." - it's useful to know if those were seen or not.</p>
<p dir="ltr">2) User has message delivered. User is requested to click a button to indicate the message has been read and understood, and will be acted upon.</p>
<p dir="ltr">This is useful in the case where a MEDEVAC request has gone through a MUC room, and it's important to see who has seen the notice and started action. Interestingly, this would need to transit MUC rooms, and have lists of users that need to acknowledge it.</p>

<p dir="ltr">My understanding is that the OP is after (1) only, but I suspect it'd be worthwhile investigating (2) as well.</p>
