Greetings Matt,
<p>How about, rather than replacing bugzilla, jabberize bugzilla.&nbsp;
Maybe so that instead of having your bugzilla installations each have their
own databases, use a jabber layer that intercepts all the mysql stuff and
redirects all data flow to your centralized database.
<p>Jabber client monitoring could all be implemented within that layer,
and with data filters, the client communications could be customized (for
example, I might only want to see activity on resolutions within a certain
component).&nbsp; Bugzilla's query logic could even be leveraged for allowing
jabber users to customize and set up the activity they want monitored.
<p>This, along with something to let me monitor CVS activity would be very
useful to me :-)
