Hi,<br><br>Multiple client connections to a single host/port are certainly possible and <br>in fact happen all the time.&nbsp; For example, a web server is usually set up<br>on port 80 and more than one client can be connected at a time.
<br><br>The underlying mechanism has nothing to do with jabber, but works something like<br>this:<br><br>A server binds a &quot;socket&quot; to a host/port (usually the local host of the server).<br>A client connects to to that host/ port and is given a connection.&nbsp; 
<br>The client's connection is in fact bound to a (host, port) pair too, but <br>the host is local to the client.&nbsp; The allocation of the client's port is usually <br>hidden from the client and the client's port number can be almost anything 
<br>(usually in the range 1000..65XXX) The communication link may then be<br>identified by the pair of (clienthost, clientport) (serverhost, server port)<br><br>If you look at documentation for sockets (man socket on unix or 
<br>look up java.net.Socket and java.net.ServerSocket in the java api<br>for example), you'll see how this is generally managed programmatically.<br><br>best,<br><br>&nbsp;  <br><br><div><span class="gmail_quote">On 9/8/06, <b class="gmail_sendername">
