No, it doesn&#39;t. It&#39;s just to store arbitrary XML data on the server and receive it later.<br>
<br>
What you want it just XMPP core, which allows you to add your own namespace and put in there whatever you like.<br>
<br>
&lt;message to=&#39;foo&#39; type=&#39;chat&#39;&gt;<br>
&lt;body&gt;My text&lt;/body&gt;<br>
&lt;somenonstandardfoo xmlns=&#39;<a href="https://nonstandard.org/foo" target="_blank">https://nonstandard.org/foo</a>&#39;&gt;<br>
&lt;foo&gt;&lt;bar/&gt;&lt;/foo&gt;<br>
&lt;baz&gt;qux!&lt;/baz&gt;<br>
&lt;/somenonstandardfoo&gt;<br>
&lt;/message&gt;<br>
<br>
You should really read XMPP core before writing an XMPP app, as this is very basic stuff, used by many of the XMPP Extensions (XEPs) ;).<br>
