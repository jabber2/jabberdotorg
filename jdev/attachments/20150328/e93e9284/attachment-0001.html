<div dir="ltr"><div><div><div>Dear all,<br></div>I would like to announce a new C#, Xamarin compatible, XMPP client library, <b>Sharp.Xmpp </b>(<a href="https://github.com/pgstath/Sharp.Xmpp">https://github.com/pgstath/Sharp.Xmpp</a>), available also as a nuget package at <a href="https://www.nuget.org/packages/Sharp.Xmpp/">https://www.nuget.org/packages/Sharp.Xmpp/</a>.<br><br>Sharp.Xmpp is a fork of the excellent but kind of dormant S22.Xmpp, with improvements, bug fixes and new features. Sharp.Xmpp is extensively tested for the <b>Xamarin Android</b>, and <b>native</b> 
.Net platforms and offers superb SOCKS5  support capabilities. <br><br></div><div>Release notes are available at <a href="https://github.com/pgstath/Sharp.Xmpp/blob/master/ReleaseNotes.md">https://github.com/pgstath/Sharp.Xmpp/blob/master/ReleaseNotes.md</a><br></div><div><br></div>Like S22.Xmpp the license of Sharp.Xmpp is MIT, so you can use it in both open source and proprietary sw projects.<br></div><div></div><div><br></div><div>Since there are not so many options of C# developers for open source C# XMPP libraries, and the S22.Xmpp seems kind of dormant, I hope this is a welcomed addition to the options available to C# developers. <br><br>In the development roadmap next priorities are OTR support and direct 
bytestream file transfers on Android and native .Net without a SOCKS5 support.<br><br>You can use the github page for any bugs, issues or requests. <br><br></div><div>Cheers,<br></div><div>Panagiotis<br></div><div><br></div></div>
