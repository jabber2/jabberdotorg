> Wow. That is a very ambitious projects.

Not really.  The main goal is to show that Jabber/XMPP is capable of
supporting all the necessary functionality for the AG, with little or no
modification.  By defining the AG as a set of protocols rather than an
implementation (as it is now, and will still be in the proposed AGv3)
anyone can include AG functionality into existing app (eg. it should be
possible to implement this as Gaim plugins).

>  What it can and can not do right now?

It primarily supports MUC-room SDP, and therefore room-shared audio and
video.  Currently it uses a small extension to the MUC protocol
(room-private data), although SPPS should make this obsolete.  There's a
corresponding patch to ejabberd for this.

> What is the license? I have not found any info about it on website

My client is GPL, but this doesn't matter, as it's not tied to my
implementation.
